## 一、仓库定位

本仓库为 EM-Core 通用智能系统中 MCC 工具执行中枢的 AI Agent 专项实现，固定 40 个核心模块，按 9 大业务集群分组。

采用认知-执行绝对解耦架构，MCC 无任何 LLM 推理与意图判断能力，只接收 ECC 下发的标准化高层目标意图，转化为数字空间底层原子系统操作。所有操作权限、风险判定、伦理约束全部由 ECC 认知层前置管控，MCC 仅做底层硬件/系统安全兜底。

顶层保留固定 40 模块，所有并发调度、故障处理、崩溃恢复逻辑全部内嵌原有模块子单元，不新增顶层模块、不破坏原有架构、不打乱编号体系。


## 二、核心架构

### 2.1 认知-执行解耦完整闭环

1. ECC-12 下发带 sign_ecc05 结构化意图报文至 CerebellumBus
2. MCC-01 校验 session、签名合法性，报文超过 30s 未入队自动丢弃并回传超时告警
3. MCC-05 完成 DAG 依赖解析，拆分可并行子任务，标记资源冲突串行任务
4. MCC-02 风险分级（与 ECC-05 完全一致的 L0/L1/L2/L3 四级体系），中高风险合并预览弹窗（超时 120s 自动拒绝）
5. 双签名放行：sign_ecc05 + sign_mcc02 齐全后任务进入可执行队列
6. 并发沙箱执行：无依赖子任务多线程并行，有依赖任务自动串行排队，全局资源锁防并发冲突
7. MCC-36 分级故障分类，可重试自动退避（最多 3 次指数退避），不可恢复触发 MCC-33 局部熔断
8. 全会话并发任务全部完成且无异常，MCC 统一上报完成信号至 ECC-12 解锁记忆锁
9. 执行日志与指标归档 MemoryBus

### 2.2 跨平台统一中间指令分层

```
MCC 上层统一抽象中间指令集（40 个核心业务模块通用，无平台耦合）
    ├── Win32 平台适配层
    ├── macOS 适配层
    ├── Android 适配层
    └── iOS 适配层
```

上层业务模块完全无平台感知，适配层统一封装 API 差异、路径格式转换、错误码映射。

### 2.3 核心特性

| 特性 | 说明 |
|------|------|
| 认知执行绝对解耦 | 零 LLM 依赖，纯状态机驱动 |
| 双签名强制安全准入 | sign_ecc05 + sign_mcc02 缺一直接丢弃 |
| 三层安全拦截 | ECC 网关分片预检 → ECC-05 语义推演 → MCC-02 底层资源锁兜底 |
| 双并发引擎 | 多 session 隔离并行 + 同 session DAG 智能并行 |
| 全局资源互斥锁 | 文件/剪贴板/窗口/网络四大锁防止并发损坏 |
| 真人操作防抖 | 贝塞尔曲线轨迹、随机延迟偏移、输入平滑滤波 |
| 多级持久化撤销链 | 并行分支独立快照栈，单分支最大深度 20，全局上限 200MB |
| 分级资源降级 | CPU>80%/内存>90%/电量<20% 自动限制并发，与 ECC 同步降级 |
| MCC 故障分级处理 | F/A/N/C/B/R/H 七大类故障自动分类，可重试自动退避 |
| 操作 WAL 预写日志 | 每次写操作前记日志，崩溃后按日志恢复或回滚 |
| 心跳卡死检测 | 每 3 秒心跳，超 10 秒判定卡死，暂停队列通知用户 |
| 断点续传 | 恢复供电后读取 WAL 日志，向 ECC-09 上报未完成 session |
| 跨平台全场景覆盖 | PC/移动端/网页/外设全适配 |


## 三、40 模块详细定义

### 集群一：顶层总控、报文接收与调度中枢（MCC-01 ~ MCC-05）

| 编号 | 模块名称 | 核心职责 |
|:---:|----------|----------|
| 01 | 总线意图接收分发单元 | 校验 session、报文格式、sign_ecc05 合法性，无签名直接拦截告警。报文超 30s 未入队自动丢弃并回传超时告警 |
| 02 | 底层安全边界闸门 | 高危黑名单永久拦截。风险分级采用与 ECC-05 完全一致的 L0/L1/L2/L3 四级体系。并行多任务合并预览弹窗，超时 120s 自动拒绝。兜底拦截 LLM 拼接恶意指令。校验通过后生成 sign_mcc02 |
| 03 | 意图解析与中间指令转换单元 | 高层意图拆分为标准化中间指令，自动标记任务资源依赖。工具能力注册预留子单元定义工具描述 schema 接口 |
| 04 | 全局执行状态机单元 | 统一管理 Idle/Running/Paused/Failed/Success 状态，支持多分支并行独立状态跟踪。心跳守护每 3 秒写入时间戳。断点续传恢复供电后读取 WAL 日志最后完成步骤 |
| 05 | 多任务队列与优先级调度单元 | DAG 依赖解析自动区分可并行/强制串行任务。全局最大并发 session=8，单 session 最大并行=6。文件/剪贴板/窗口/网络四大全局互斥锁。API 限流自动退避重试最多 3 次 |

| 07 | 输入平滑滤波防抖单元 | 全局输入预处理，随机延迟偏移适配并行输入场景 |
| 08 | 剪贴板与数据脱敏单元 | 剪贴板全局互斥锁，多任务读写排队，自动脱敏隐私明文 |
| 09 | 跨应用 IPC 与数据总线单元 | 进程间通信桥接，跨应用传输资源锁防抢占冲突 |
| 10 | 弹窗与 UI 异常自动处理单元 | 批量并行操作弹窗合并降噪。系统权限弹窗不可自动处理，上报 ECC-10 请求用户介入 |

### 集群六：安全审计、撤销回滚、熔断防护集群（MCC-30 ~ MCC-34）

| 编号 | 模块名称 | 核心职责 |
|:---:|----------|----------|

| 31 | 多级 Undo/Redo 持久化撤销单元 | 并行分支独立快照栈，单分支最大深度 20，全局上限 200MB。支持单分支、多分支批量、整 session 整体回滚。操作 WAL 预写日志子单元在每次写操作前记录快照 |

### 集群七：执行结果校验、记忆联动上报集群（MCC-35 ~ MCC-36）

| 编号 | 模块名称 | 核心职责 |
|:---:|----------|----------|
| 35 | 执行结果校验单元 | 并行多任务结果聚合校验，批量统计成功/失败/超时/熔断分支 |
| 36 | ECC 与 MLNF 双向上报单元 | 等待全会话所有并行分支执行完毕后统一上报。七大类故障（F/A/N/C/B/R/H）自动分类，可重试自动退避最多 3 次指数退避，需用户介入经 ECC-10 推送，不可恢复触发 MCC-33 局部熔断 |

**标准化故障回执格式：**

```json
{
  "fault_code": "F1-404",
  "fault_category": "FILE_NOT_FOUND",
  "severity": "MEDIUM",
  "session_id": "session_x",
  "task_id": "task_y",
  "context_snapshot": {
    "target_path": "/home/user/report.md",
    "last_known_location": "/home/user/downloads/report.md"
  },
  "suggested_action": "ASK_USER_ALTERNATIVE",
  "timestamp": "2026-06-22T14:31:05.000Z"
}
```

**七大类故障码：**

| 类别 | 前缀 | 典型场景 |
|------|:---:|------|
| F 文件 | F1-F7 | 文件不存在/权限不足/被占用/磁盘满 |
| A 应用 | A1-A6 | 应用未安装/未响应/控件定位失败 |
| N 网络 | N1-N6 | 网络不可用/API超时/限流/认证失效 |
| C 命令 | C1-C5 | 命令不存在/被安全拦截/执行超时 |
| B 剪贴板 | B1-B3 | 剪贴板为空/格式不匹配/脱敏拦截 |
| R 并发 | R1-R4 | 锁等待超时/队列满/会话数满 |
| H 硬件 | H1-H3 | 外设未连接/电池不足/存储介质弹出 |

## 四、全局锁定规范

| 规则编号 | 内容 |
|:---:|------|
| 1 | MCC 内部 01–40 模块编号连续永久固定，同版本不增删、不重排。所有并发、故障处理、崩溃恢复均以内置子单元实现 |
| 2 | 所有写操作必须经 MCC-02 双签名校验放行，sign_ecc05 由 ECC-05 生成，sign_mcc02 由 MCC-02 生成 |
| 3 | 全局仅 CerebellumBus + MemoryBus 双总线通信，禁止跨模块直连底层 |
| 4 | 记忆锁由 ECC-12 全权管控，MCC 仅负责全会话并发任务收尾回执 |
| 5 | 多 session 天然完全隔离并行。同 session 严格 DAG 依赖调度。四大资源永久全局互斥锁 |
| 6 | 并行任务必须分组快照、分支独立撤销。单分支快照最大深度 20，全局撤销栈总容量 200MB |
| 7 | 所有执行异常必须经 MCC-36 分级故障分类器标准化上报，可重试自动退避最多 3 次，不可恢复自动触发 MCC-33 局部熔断 |
| 8 | 所有写操作执行前必须写入 WAL 预写日志。系统恢复供电后 MCC-04 自动读取日志并上报 ECC-09 |
