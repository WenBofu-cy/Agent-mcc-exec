# ag-mcc-03-资源配额管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-03 |
| 模块名称 | 资源配额管控单元 |
| 所属分区 | 一、执行中枢调度 |
| 核心职责 | 监控并管控行动执行层各执行模块的资源消耗，包括 API 调用频次、代码执行内存上限、文件操作路径白名单、大模型 Token 配额等。在执行任务前，校验当前资源使用是否在安全阈值内——若资源充足则放行；若接近上限则触发限流；若已耗尽则拒绝执行并返回资源不足通知。同时向 ag-mcc-12（执行日志记录单元）周期性上报资源使用状态。不参与工具的实际调用，仅负责资源配额的校验与管控 |
| 依赖模块 | ag-mcc-01（执行调度核心，在指令分发前查询资源配额）、ag-mcc-04（工具注册中心，获取各工具的预设资源配额）、ag-mcc-12（执行日志记录单元，上报资源使用状态） |
| 被依赖模块 | ag-mcc-01（接收配额校验结果）、ag-mcc-06~08（各执行模块，接收资源约束参数） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常放行 | `NORMAL_FLOW` | 所有资源类型均未触及预警阈值，正常放行任务 | 系统初始化完成，资源使用率 < 70% |
| 资源预警 | `RESOURCE_WARNING` | 至少一项资源使用率超过预警阈值（≥ 70%），触发温和限流 | 任一资源使用率 ≥ 70% |
| 资源紧急 | `RESOURCE_CRITICAL` | 至少一项资源使用率超过紧急阈值（≥ 90%），暂停新任务并强制限流 | 任一资源使用率 ≥ 90% |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 配额校验请求 | Struct（指令ID + 工具名称 + 工具类型 + 预估资源消耗：API次数/内存MB/存储KB/Token数 + 优先级） | ag-mcc-01 执行调度核心（通过内部调度总线） | 每次指令分发前 | **高** |
| 工具资源配额预设 | Struct（工具名称 + 单次调用最大API次数 + 单次最大内存MB + 单次最大存储KB + 单次最大Token数） | ag-mcc-04 工具注册中心 | 系统初始化加载，配置更新时 | 普通 |
| 当前资源使用量快照 | Struct（今日API调用次数 + 当前总内存使用MB + 当前总存储使用KB + 今日Token消耗） | 系统资源监控接口 | 周期性（每5秒）或配额查询时 | **高** |
| 资源释放通知 | Struct（指令ID + 释放的API次数 + 释放的内存MB + 释放的Token数） | ag-mcc-06/07/08 各执行模块 | 任务执行完成时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 配额校验结果 | Struct（指令ID + 是否放行 + 拒绝原因 + 资源约束参数：限速/限内存/限Token） | ag-mcc-01 执行调度核心 | 配额校验完成时 | **高** |
| 资源限流通知 | Struct（限流类型 + 当前使用量 + 配额上限 + 预计恢复时间 + 受影响模块列表） | ag-mcc-01（广播至各执行模块） | 触发限流时 | **高** |
| 资源使用状态上报 | Struct（当前状态 + 各资源使用率 + 今日配额消耗 + 剩余配额） | ag-mcc-12 执行日志记录单元 | 周期性（每30秒）或状态变更时 | 普通 |


## 资源配额配置

### 一、各类资源阈值

| 资源类型 | 单位 | 默认配额上限 | 预警阈值 | 紧急阈值 | 重置周期 |
|----------|:---:|:---:|:---:|:---:|:---:|
| API 调用频次 | 次/分钟 | 100 | 70% (70次) | 90% (90次) | 每分钟重置 |
| 代码执行内存 | MB | 512 | 70% (358MB) | 90% (460MB) | 实时（任务结束后释放） |
| 文件操作存储 | MB | 100 | 70% (70MB) | 90% (90MB) | 实时（任务结束后释放） |
| 大模型 Token | Token/天 | 100,000 | 70% (70,000) | 90% (90,000) | 每日零点重置 |
| 并发连接数 | 个 | 10 | 7 | 9 | 实时（连接关闭后释放） |

### 二、限流策略

| 资源状态 | 处理方式 |
|----------|------|
| 正常（< 70%） | 直接放行，不附加任何限制 |
| 预警（70% – 90%） | 放行，但附加限速参数：API调用间隔 ≥ 500ms，内存上限降至配额 80% |
| 紧急（≥ 90%） | 拒绝新任务，仅允许高优先级（用户显式标注为"紧急"）的任务执行 |
| 已耗尽（100%） | 拒绝一切任务，返回"资源已耗尽"通知 |


## 核心处理逻辑

```
FUNCTION quota_controller_main_loop():
    STATE_NORMAL = NORMAL_FLOW
    STATE_WARN = RESOURCE_WARNING
    STATE_CRITICAL = RESOURCE_CRITICAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化资源使用计数器 = {API: 0, 内存: 0, 存储: 0, Token: 0, 并发: 0}
    加载各工具资源配额预设
    初始化配额上限配置

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：更新资源使用量快照
        IF 收到当前资源使用量快照:
            更新资源使用计数器 = 快照
            重新评估资源状态()

        // 第3步：处理资源释放通知
        IF 收到资源释放通知:
            资源使用计数器[API] -= 释放.API次数
            资源使用计数器[内存] -= 释放.内存MB
            资源使用计数器[Token] -= 释放.Token数
            重新评估资源状态()

        // 第4步：处理配额校验请求
        IF 收到配额校验请求:
            // 紧急状态下仅放行高优先级任务
            IF internal_state == STATE_CRITICAL:
                IF 请求.优先级 != HIGH:
                    向 ag-mcc-01 返回配额校验结果(放行=False, 拒绝原因="资源紧急，仅允许高优先级任务")
                    CONTINUE

            // 已耗尽状态拒绝一切
            IF 任一资源 >= 配额上限:
                向 ag-mcc-01 返回配额校验结果(放行=False, 拒绝原因="资源已耗尽")
                CONTINUE

            // 获取该工具的配额预设
            工具配额 = 获取工具资源配额预设(请求.工具名称)
            IF 工具配额 == None:
                工具配额 = 默认配额

            // 逐项校验
            拒绝原因列表 = []
            约束参数 = {}

            IF 资源使用计数器[API] + 工具配额.API次数 > 配额上限.API:
                拒绝原因列表.append("API配额不足")

            IF 资源使用计数器[内存] + 工具配额.内存MB > 配额上限.内存:
                IF internal_state == STATE_WARN:
                    约束参数[内存上限] = 配额上限.内存 * 0.8
                ELSE:
                    拒绝原因列表.append("内存配额不足")

            IF 资源使用计数器[Token] + 工具配额.Token数 > 配额上限.Token:
                拒绝原因列表.append("Token配额不足")

            IF 资源使用计数器[并发] >= 配额上限.并发:
                拒绝原因列表.append("并发连接数已满")

            // 返回校验结果
            IF 拒绝原因列表为空:
                向 ag-mcc-01 返回配额校验结果(放行=True, 约束参数=约束参数)
            ELSE:
                向 ag-mcc-01 返回配额校验结果(放行=False, 拒绝原因=JOIN(拒绝原因列表, "; "))

        // 第5步：周期性资源状态上报
        IF 距上次状态上报 >= 30秒:
            向 ag-mcc-12 上报资源使用状态

        SLEEP 50ms


FUNCTION 重新评估资源状态():
    IF 任一资源使用率 >= 90%:
        SET internal_state = STATE_CRITICAL
        向 ag-mcc-01 发送资源限流通知("紧急", 当前使用量)
    ELSE IF 任一资源使用率 >= 70% AND internal_state == STATE_NORMAL:
        SET internal_state = STATE_WARN
        向 ag-mcc-01 发送资源限流通知("预警", 当前使用量)
    ELSE IF 所有资源使用率 < 70% AND internal_state != STATE_NORMAL:
        SET internal_state = STATE_NORMAL
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 工具无预设配额配置 | 使用默认配额（API=10次/分钟, 内存=64MB, Token=1000） | 该工具的配额被注册 |
| 资源使用量快照中断（超过 30 秒无更新） | 使用最后一次有效值，标记"数据可能过时"，将状态临时设为 WARN | 快照恢复更新 |
| 资源释放通知中释放量超过当前使用量 | 使用计数器归零（不出现负数），记录异常日志 | — |
| 同一指令重复发起配额校验 | 基于指令ID去重，5秒内相同指令ID的校验直接返回缓存结果 | — |
| 紧急熔断 | 暂停所有配额校验，保持当前计数器状态 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| Q-01 | 资源配额管控单元仅负责校验与通知，不得直接终止或中断正在执行的任务 |
| Q-02 | 资源使用计数器在系统重启后必须重新从零开始计数，防止残留数据导致误杀 |
| Q-03 | 紧急状态下拒绝新任务时，必须明确告知拒绝原因与资源现状 |
| Q-04 | 资源使用状态数据仅用于内部管控，不得泄露至外部系统或第三方模块 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M03-01 | `NORMAL_FLOW`，资源充足 | 配额校验请求（API调用，预估1次） | 放行=True，无拒绝原因 |
| TC-M03-02 | `RESOURCE_WARNING`，API接近上限 | 配额校验请求（API调用，预估1次，当前API使用率=75%） | 放行=True，附带限速约束参数 |
| TC-M03-03 | `RESOURCE_CRITICAL`，内存已满 | 配额校验请求（代码执行，内存需求=100MB，当前内存使用率=92%） | 放行=False，拒绝原因="内存配额不足" |
| TC-M03-04 | `RESOURCE_CRITICAL`，高优先级任务放行 | 配额校验请求（优先级=HIGH，代码执行，当前资源紧急） | 放行=True，紧急状态下高优先级例外 |
| TC-M03-05 | `NORMAL_FLOW`，并发连接数已满 | 配额校验请求（当前并发数=10） | 放行=False，拒绝原因="并发连接数已满" |
| TC-M03-06 | `SYSTEM_PAUSED`，收到校验请求 | 配额校验请求 | 拒绝处理，保持暂停状态 |
