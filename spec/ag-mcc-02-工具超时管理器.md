# ag-mcc-02-工具超时管理器 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-02 |
| 模块名称 | 工具超时管理器 |
| 所属分区 | 一、执行中枢调度 |
| 核心职责 | 监控所有活跃的工具调用任务的执行时长，在任务超过预设的超时时间后自动触发中断流程。管理各类型工具的差异化超时阈值，支持任务级别的超时动态调整。当检测到超时事件时，立即通知 ag-mcc-01（执行调度核心）强制终止对应任务，并向 ag-mcc-11（闭环反馈单元）上报超时事件供审计。不参与工具的实际调用或参数校验，仅负责时间监控与超时告警 |
| 依赖模块 | ag-mcc-01（执行调度核心，获取活跃任务列表及其超时配置）、ag-mcc-11（闭环反馈单元，上报超时事件） |
| 被依赖模块 | ag-mcc-01（接收超时通知并强制终止任务） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 系统运行正常，定时扫描活跃任务 | 系统初始化完成 |
| 超时检测 | `TIMEOUT_DETECTED` | 检测到至少一个任务已超时，正在执行中断通知 | 任一任务的已执行时间超过其超时阈值 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 活跃任务快照 | Struct（任务ID + 任务类型 + 开始时间戳 + 超时阈值 + 是否允许延期） | ag-mcc-01 执行调度核心（通过内部调度总线） | 周期性（每500ms）或任务状态变更时 | **高** |
| 超时阈值配置 | Struct（工具类型 → 默认超时秒数 + 最大超时秒数 + 是否允许动态调整） | ag-mcc-04 工具注册中心 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 超时通知 | Struct（任务ID + 已执行时长 + 超时阈值 + 超时原因） | ag-mcc-01 执行调度核心 | 检测到任务超时时 | **高** |
| 超时事件日志 | Struct（任务ID + 工具名称 + 超时阈值 + 实际耗时 + 时间戳） | ag-mcc-11 闭环反馈单元 | 每次超时事件发生时 | 普通 |
| 超时统计上报 | Struct（当前状态 + 活跃监控任务数 + 今日超时次数 + 超时率） | ag-mcc-01 执行调度核心 | 周期性（每60秒）或状态变更时 | 普通 |


## 超时阈值配置

### 一、按工具类型的默认超时

| 工具类型 | 默认超时（秒） | 最大超时（秒） | 是否允许动态调整 | 说明 |
|:---:|:---:|:---:|:---:|------|
| API 调用 | 60 | 300 | 是 | 依赖外部服务的 API 请求 |
| 代码执行 | 30 | 120 | 是 | 沙箱中执行的用户代码 |
| 文件操作 | 10 | 30 | 否 | 本地文件读写，通常耗时较短 |
| 大模型调用 | 120 | 600 | 是 | 大模型推理可能较慢 |
| 数据库操作 | 15 | 60 | 是 | 数据库查询或写入 |

### 二、动态调整策略

| 调整触发条件 | 调整方式 | 有效期 |
|----------|------|:---:|
| 该工具的历史平均耗时超过默认超时 80% | 将超时阈值上调至历史平均耗时 × 1.2 | 单次任务生效 |
| 用户显式标注为"允许长时间等待" | 将超时阈值设为最大超时 | 当前会话有效 |
| 系统资源紧张（CPU > 85%） | 将超时阈值下调 20% | 资源紧张期间有效 |


## 核心处理逻辑

```
FUNCTION timeout_manager_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_TIMEOUT = TIMEOUT_DETECTED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化活跃任务监控表 = {}  // 任务ID → {开始时间, 超时阈值, 工具类型}
    加载超时阈值配置

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收活跃任务快照
        IF 收到 ag-mcc-01 活跃任务快照:
            // 更新监控表
            FOR EACH 任务 IN 快照:
                IF 任务.任务ID NOT IN 活跃任务监控表:
                    活跃任务监控表[任务.任务ID] = {
                        开始时间: 任务.开始时间戳,
                        超时阈值: 任务.超时阈值,
                        工具类型: 任务.工具类型
                    }
                ELSE:
                    更新活跃任务监控表[任务.任务ID].超时阈值 = 任务.超时阈值

            // 清理已结束的任务
            FOR EACH 任务ID IN 活跃任务监控表:
                IF 任务ID NOT IN 快照.任务ID列表:
                    从活跃任务监控表中移除任务ID

        // 第3步：定时扫描超时（每500ms）
        IF 距上次扫描 >= 500ms:
            当前时间 = NOW()
            超时任务列表 = []

            FOR EACH 任务ID, 监控信息 IN 活跃任务监控表:
                已执行时长 = 当前时间 - 监控信息.开始时间
                IF 已执行时长 >= 监控信息.超时阈值:
                    超时任务列表.append({
                        任务ID: 任务ID,
                        已执行时长: 已执行时长,
                        超时阈值: 监控信息.超时阈值,
                        工具类型: 监控信息.工具类型
                    })

            IF 超时任务列表非空:
                SET internal_state = STATE_TIMEOUT

                FOR EACH 超时任务 IN 超时任务列表:
                    // 通知执行调度核心强制终止
                    向 ag-mcc-01 发送超时通知(超时任务)

                    // 记录超时事件日志
                    向 ag-mcc-11 发送超时事件日志(超时任务)

                    // 从监控表中移除已超时任务
                    从活跃任务监控表中移除(超时任务.任务ID)

                SET internal_state = STATE_NORMAL

        // 第4步：周期性统计上报
        IF 距上次统计上报 >= 60秒:
            向 ag-mcc-01 上报超时统计

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 活跃任务快照中某任务的开始时间为未来时间 | 视为数据异常，使用当前时间替换，记录异常日志 | 数据修正 |
| 工具类型不在已知配置中 | 使用默认超时 30 秒，标记"未知类型默认值" | 工具类型被注册 |
| 同一任务重复触发超时通知 | 去重检查：已从监控表移除的任务不再发送超时通知 | — |
| 超时阈值动态调整后低于 5 秒 | 强制将阈值设为 5 秒（最小安全值） | — |
| 紧急熔断 | 清空监控表，暂停所有扫描，保留当前统计值 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 活跃任务快照 | 只读 | ag-mcc-01 发送 |
| 内部调度总线 | 读 | 超时阈值配置 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 写 | 超时通知 | 专属写入 | 向 ag-mcc-01 发送 |
| 内部调度总线 | 写 | 超时事件日志 | 专属写入 | 向 ag-mcc-11 发送 |
| 内部调度总线 | 写 | 超时统计上报 | 周期性写入 | 向 ag-mcc-01 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| T-01 | 超时管理器仅负责时间监控与通知，不得直接执行任务的中断或资源回收 |
| T-02 | 最小超时阈值硬编码为 5 秒，任何动态调整不得低于此值，防止误杀快速任务 |
| T-03 | 超时事件日志必须包含完整的任务ID、工具名称、超时阈值与实际耗时，不可篡改 |
| T-04 | 紧急熔断时监控表必须清空，避免恢复后残留过期的监控数据导致误报 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M02-01 | `NORMAL_MONITOR`，任务超时 | 活跃任务快照（任务A，已执行65秒，超时阈值60秒） | 发送超时通知至 ag-mcc-01，记录超时事件日志 |
| TC-M02-02 | `NORMAL_MONITOR`，任务未超时 | 活跃任务快照（任务B，已执行30秒，超时阈值60秒） | 不触发超时，继续监控 |
| TC-M02-03 | `NORMAL_MONITOR`，任务完成自动清理 | 活跃任务快照（不含任务C），但监控表中有任务C | 从监控表中移除任务C |
| TC-M02-04 | `NORMAL_MONITOR`，未知工具类型 | 活跃任务快照（任务D，工具类型=UNKNOWN，无对应阈值配置） | 使用默认超时 30 秒 |
| TC-M02-05 | `NORMAL_MONITOR`，动态调整后低于最小阈值 | 计算出的新阈值 = 3 秒 | 强制设为 5 秒 |
| TC-M02-06 | `SYSTEM_PAUSED`，收到活跃任务快照 | 活跃任务快照 | 拒绝处理，保持暂停状态 |
