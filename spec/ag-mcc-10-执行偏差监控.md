# ag-mcc-10-执行偏差监控 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-10 |
| 模块名称 | 执行偏差监控 |
| 所属分区 | 四、反馈与日志 |
| 核心职责 | 在工具调用执行完成后，将实际执行结果与 ag-ecc-02（任务规划模块）下发的预期目标进行量化比对。监控执行耗时偏差、输出数据偏差、资源消耗偏差等多维度指标，判定执行质量是否符合预期。当偏差超过预设阈值时，生成偏差告警通知 ag-mcc-11（闭环反馈单元），为 ECC 认知大脑的任务恢复决策提供数据支撑。不参与任务规划或结果修正，仅负责偏差的客观量化与告警 |
| 依赖模块 | ag-mcc-01（执行调度核心，提供待分析的执行结果）、ag-mcc-09（结果校验器，提供已校验的结果数据）、ag-ecc-02（任务规划模块，提供预期目标定义） |
| 被依赖模块 | ag-mcc-11（闭环反馈单元，接收偏差分析报告与告警）、ag-mcc-12（执行日志记录单元，记录偏差事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待结果 | `WAITING_RESULT` | 无待分析的执行结果 | 系统初始化完成，分析队列为空 |
| 偏差分析中 | `ANALYZING` | 正在将实际结果与预期目标进行多维度量化比对 | 收到待分析的执行结果 |
| 分析完成 | `ANALYZED` | 偏差分析完成，已生成分析报告 | 所有维度比对完毕 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

|--------|----------|----------|----------|:---:|

## 偏差监控维度

### 一、量化比对维度

| 监控维度 | 权重 | 偏差计算方式 | 告警阈值 | 数据来源 |
|----------|:---:|------|:---:|------|
| 耗时偏差 | 0.25 | abs(实际耗时 - 预估耗时) / 预估耗时 | > 50% | 实际耗时 vs 预估耗时 |
| 输出格式偏差 | 0.30 | 1 - (匹配的字段数 / 预期字段总数) | > 30% | 实际输出 vs 预期输出格式 |
| 关键指标偏差 | 0.25 | 根据 KPI 类型计算（数值型/布尔型/枚举型） | KPI 未达成 | 实际KPI值 vs 目标KPI值 |
| 资源消耗偏差 | 0.20 | abs(实际资源 - 预估资源) / 预估资源 | > 40% | 实际资源 vs 预估资源 |

### 二、综合偏差度计算

```
综合偏差度 = 0.25 × 耗时偏差率 + 0.30 × 输出格式偏差率 + 0.25 × KPI偏差率 + 0.20 × 资源消耗偏差率

告警触发条件：
- 综合偏差度 ≥ 0.5 → 严重告警
- 综合偏差度 ≥ 0.3 且 < 0.5 → 一般告警
- 综合偏差度 < 0.3 → 正常，记录日志不告警
```

### 三、各工具类型的KPI定义

| 工具类型 | KPI类型 | KPI示例 | 判定方式 |
|:---:|:---:|------|------|
| API调用 | 数值型/布尔型 | 返回结果数量 ≥ N, 状态码 = 200 | 数值比较/布尔判断 |
| 代码执行 | 数值型 | 退出码 = 0, 执行时间 ≤ 预估 | 数值比较 |
| 文件操作 | 布尔型 | 文件是否成功创建/删除, 哈希是否匹配 | 布尔判断 |
| 大模型调用 | 数值型 | Token消耗 ≤ 预估, 响应长度 ≥ N | 数值比较 |


## 核心处理逻辑

```
FUNCTION deviation_monitor_main_loop():
    STATE_WAIT = WAITING_RESULT
    STATE_ANALYZE = ANALYZING
    STATE_DONE = ANALYZED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化监控统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收校验后执行结果
        IF 收到校验后执行结果:
            SET internal_state = STATE_ANALYZE
            结果 = 校验后执行结果
            偏差详情 = {}
            告警维度列表 = []

            // 2a. 获取预期目标定义
            预期目标 = 向 ag-ecc-02 查询预期目标定义(结果.步骤ID)
            IF 预期目标 == None:
                // 无预期目标时使用默认宽松标准
                预期目标 = 默认预期目标

            // 2b. 耗时偏差分析
            IF 结果.实际耗时 != None AND 预期目标.预估耗时 != None:
                耗时偏差率 = abs(结果.实际耗时 - 预期目标.预估耗时) / 预期目标.预估耗时
                偏差详情["耗时偏差"] = round(耗时偏差率, 3)
                IF 耗时偏差率 > 0.5:
                    告警维度列表.append("耗时偏差过大")

            // 2c. 输出格式偏差分析
            IF 预期目标.预期输出格式 != None:
                匹配字段数 = 计算匹配字段数(结果.清洗后输出数据, 预期目标.预期输出格式)
                预期字段总数 = len(预期目标.预期输出格式.字段列表)
                格式偏差率 = 1 - (匹配字段数 / 预期字段总数) if 预期字段总数 > 0 else 0
                偏差详情["输出格式偏差"] = round(格式偏差率, 3)
                IF 格式偏差率 > 0.3:
                    告警维度列表.append("输出格式偏差过大")

            // 2d. 关键指标偏差分析
            IF 预期目标.关键成功指标 != None:
                KPI达成 = 检查KPI达成情况(结果.清洗后输出数据, 预期目标.关键成功指标)
                偏差详情["KPI偏差"] = KPI达成
                IF NOT KPI达成.全部通过:
                    告警维度列表.append("KPI未达成")

            // 2e. 资源消耗偏差分析
            IF 结果.实际资源消耗 != None AND 预期目标.最大允许资源消耗 != None:
                资源偏差率 = 计算资源偏差率(结果.实际资源消耗, 预期目标.最大允许资源消耗)
                偏差详情["资源消耗偏差"] = round(资源偏差率, 3)
                IF 资源偏差率 > 0.4:
                    告警维度列表.append("资源消耗偏差过大")

            // 2f. 计算综合偏差度
            综合偏差度 = 计算综合偏差度(偏差详情)

            // 2g. 判定告警等级
            IF 综合偏差度 >= 0.5:
                告警等级 = "严重"
            ELSE IF 综合偏差度 >= 0.3:
                告警等级 = "一般"
            ELSE:
                告警等级 = "正常"

            // 第3步：生成偏差分析报告
            SET internal_state = STATE_DONE

            分析报告 = 构建偏差分析报告(
                指令ID=结果.指令ID,
                步骤ID=结果.步骤ID,
                计划ID=结果.计划ID,
                综合偏差度=round(综合偏差度, 3),
                偏差详情=偏差详情,
                告警等级=告警等级,
                建议处理=生成处理建议(告警等级, 告警维度列表)
            )

            // 发送至闭环反馈单元
            向 ag-mcc-11 发送偏差分析报告(分析报告)

            // 触发告警时发送偏差告警
            IF 告警等级 != "正常":
                向 ag-mcc-11 发送偏差告警(
                    指令ID=结果.指令ID,
                    步骤ID=结果.步骤ID,
                    偏差维度=告警维度列表,
                    偏差幅度=综合偏差度,
                    严重等级=告警等级
                )

            // 记录偏差事件日志
            向 ag-mcc-12 发送偏差事件日志(
                指令ID=结果.指令ID,
                偏差类型=告警等级,
                偏差度=综合偏差度
            )

            // 更新统计
            更新监控统计(告警等级, 综合偏差度)

            SET internal_state = STATE_WAIT

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报监控状态

        SLEEP 10ms


FUNCTION 计算综合偏差度(偏差详情):
    耗时偏差 = 偏差详情.GET("耗时偏差", 0)
    格式偏差 = 偏差详情.GET("输出格式偏差", 0)
    KPI偏差 = IF 偏差详情.GET("KPI偏差") != None AND NOT 偏差详情["KPI偏差"].全部通过 THEN 1.0 ELSE 0
    资源偏差 = 偏差详情.GET("资源消耗偏差", 0)

    RETURN round(0.25 * 耗时偏差 + 0.30 * 格式偏差 + 0.25 * KPI偏差 + 0.20 * 资源偏差, 3)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 预期目标定义缺失（任务规划模块未提供） | 使用默认宽松标准：耗时偏差阈值=100%，格式偏差阈值=50%，KPI不检查 | ag-ecc-02 补充预期定义 |
| 实际执行结果中缺少关键字段 | 该维度偏差率记为 1.0（最大偏差），标记"数据不完整" | 执行模块返回完整数据 |
| 预期目标中的KPI类型无法识别 | 跳过KPI偏差分析，仅分析耗时、格式、资源三个维度 | KPI类型被注册 |
| 执行结果为失败/超时状态 | 直接标记综合偏差度=1.0，告警等级="严重"，跳过细节分析 | — |
| 同一指令重复分析 | 基于指令ID去重，5秒内相同指令ID直接返回缓存结果 | — |
| 紧急熔断 | 暂停分析，未完成的请求丢弃 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

| 内部调度总线 | 写 | 偏差事件日志 | 专属写入 | 向 ag-mcc-12 发送 |
| 内部调度总线 | 写 | 监控状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| D-01 | 本模块仅做偏差分析与告警，不直接干预任务执行或结果修正 |
| D-02 | 偏差分析过程中使用的预期目标定义不得包含任何用户个人身份信息或敏感数据 |
| D-03 | 偏差告警仅作为辅助决策参考，最终恢复策略由 ag-ecc-02 任务规划模块决定 |
| D-04 | 偏差分析报告中的原始输出数据必须在发送前经过去敏感化处理 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M10-01 | `WAITING_RESULT`，正常偏差 | 执行结果（实际耗时=8s，预估=10s，格式完全匹配，KPI达成） | 综合偏差度<0.3，告警等级=正常 |
| TC-M10-02 | `WAITING_RESULT`，耗时严重超标 | 执行结果（实际耗时=25s，预估=10s，其他正常） | 耗时偏差率=1.5>0.5，触发告警，告警维度含"耗时偏差过大" |
| TC-M10-03 | `WAITING_RESULT`，KPI未达成 | 执行结果（实际KPI不满足目标值） | KPI偏差=1.0，综合偏差度≥0.25，触发告警 |
| TC-M10-04 | `WAITING_RESULT`，多维度综合告警 | 执行结果（耗时超标+格式不匹配+资源超标） | 综合偏差度≥0.5，告警等级=严重 |
| TC-M10-05 | `WAITING_RESULT`，执行失败 | 执行结果（状态=失败） | 直接标记综合偏差度=1.0，告警等级=严重 |
| TC-M10-06 | `WAITING_RESULT`，无预期目标定义 | 执行结果（正常，但无预期目标） | 使用默认宽松标准分析，标记"无预期目标" |
