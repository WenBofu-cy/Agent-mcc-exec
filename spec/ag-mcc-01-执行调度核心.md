# ag-mcc-01-执行调度核心 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-01 |
| 模块名称 | 执行调度核心 |
| 所属分区 | 一、执行中枢调度 |
| 核心职责 | 作为 Agent-mcc-exec 行动执行层的总调度入口，接收 ECC 认知大脑（通过 CerebellumBus）下发的工具调用指令，统一分发至各执行模块（API调用引擎、代码执行沙箱、文件操作执行器等）。汇总各执行模块返回的结果，形成结构化闭环回执上报至 ECC。管理执行超时、并发控制与执行优先级。不参与任何任务决策，仅负责任务指令的接收、分发与结果汇总 |
| 依赖模块 | ag-ecc-12（资源调度模块，下发工具调用指令）、ag-mcc-02（工具超时管理器）、ag-mcc-04（工具注册中心）、ag-mcc-06（API调用引擎）、ag-mcc-07（代码执行沙箱）、ag-mcc-08（文件操作执行器） |
| 被依赖模块 | ag-ecc-12（接收执行回执）、ag-mcc-02~12（接收调度指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无活跃任务，等待 ECC 下发新指令 | 系统初始化完成，活跃任务数为 0 且队列为空 |
| 指令分发中 | `DISPATCHING` | 正在将收到的指令按工具类型路由至对应执行模块 | 收到新的工具调用指令 |
| 等待结果 | `WAITING_RESULTS` | 指令已分发，正在等待各执行模块返回结果 | 指令分发完成，存在活跃任务 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 工具调用指令 | Struct（指令ID + 步骤ID + 计划ID + 工具名称 + 工具类型：API/CODE/FILE + 参数列表 + 执行优先级 + 超时时间 + 安全令牌） | ag-ecc-12 资源调度模块（通过 CerebellumBus） | ECC 决策完成，需要执行具体工具调用时 | **高** |
| API 调用执行结果 | Struct（指令ID + 执行状态 + 输出数据 + 错误码 + 耗时 + 资源消耗） | ag-mcc-06 API调用引擎 | API 调用完成时 | **高** |
| 代码执行结果 | Struct（指令ID + 执行状态 + 输出数据 + 错误码 + 耗时 + 资源消耗） | ag-mcc-07 代码执行沙箱 | 代码执行完成时 | **高** |
| 文件操作结果 | Struct（指令ID + 执行状态 + 输出数据 + 错误码 + 耗时） | ag-mcc-08 文件操作执行器 | 文件操作完成时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 执行回执 | Struct（指令ID + 步骤ID + 计划ID + 执行状态：成功/失败/超时/异常 + 输出数据 + 错误码 + 耗时 + 资源消耗） | ag-ecc-12 资源调度模块（通过 CerebellumBus） | 每个指令执行完成时 | **高** |
| API 调用分发指令 | Struct（指令ID + 工具名称 + 参数列表 + 超时时间 + 安全令牌） | ag-mcc-06 API调用引擎 | 指令类型为 API 时 | **高** |
| 代码执行分发指令 | Struct（指令ID + 代码内容 + 执行环境 + 超时时间 + 安全令牌） | ag-mcc-07 代码执行沙箱 | 指令类型为 CODE 时 | **高** |
| 文件操作分发指令 | Struct（指令ID + 操作类型 + 文件路径 + 数据内容 + 安全令牌） | ag-mcc-08 文件操作执行器 | 指令类型为 FILE 时 | **高** |
| 执行状态上报 | Struct（当前状态 + 活跃任务数 + 队列深度 + 总执行次数 + 成功率） | ag-ecc-12 资源调度模块 | 周期性（每30秒）或状态变更时 | 普通 |


## 工具类型与分发路由

| 工具类型 | 标识 | 目标执行模块 | 典型指令示例 |
|:---:|:---:|------|------|
| API 调用 | `API` | ag-mcc-06 API调用引擎 | 调用天气查询 API、发送 HTTP 请求 |
| 代码执行 | `CODE` | ag-mcc-07 代码执行沙箱 | 执行 Python 脚本、运行用户自定义代码 |
| 文件操作 | `FILE` | ag-mcc-08 文件操作执行器 | 读取文件、写入文件、删除文件 |

## 并发控制策略

| 规则项 | 默认值 | 说明 |
|--------|:---:|------|
| 最大并发执行数 | **10** | 同时最多执行的工具调用任务数 |
| 队列最大长度 | **100** | 超出此数量的指令将被拒绝 |
| 高优先级抢占 | 启用 | HIGH 优先级任务可优先从队列中取出执行 |
| 单任务最大超时 | 300 秒 | 超出此时间的任务将被强制中断 |


## 核心处理逻辑

```
FUNCTION execution_core_main_loop():
    STATE_IDLE = IDLE
    STATE_DISPATCH = DISPATCHING
    STATE_WAIT = WAITING_RESULTS
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化活跃任务表 = {}  // 指令ID → 任务上下文
    初始化指令队列 = []
    初始化执行统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：收集各执行模块返回的结果
        IF internal_state == STATE_WAIT:
            // 轮询检查已完成的任务
            FOR EACH 活跃任务ID IN 活跃任务表:
                结果 = 查询对应执行模块的结果(活跃任务ID)
                IF 结果 != None:
                    汇总执行回执 = 构建结构化闭环回执(结果)
                    向 ag-ecc-12 发送执行回执(汇总执行回执)
                    从活跃任务表中移除(活跃任务ID)
                    更新统计计数器

            IF 活跃任务表为空 AND 指令队列为空:
                SET internal_state = STATE_IDLE

        // 第3步：从队列中取出等待的任务（如果并发数允许）
        IF 指令队列非空 AND 活跃任务表.size < 最大并发数:
            下一指令 = 按优先级从队列取出(指令队列)
            SET internal_state = STATE_DISPATCH
            执行指令分发(下一指令)

        // 第4步：接收新指令
        IF 收到新的工具调用指令:
            IF 活跃任务表.size >= 最大并发数:
                IF 指令队列.size < 队列最大长度:
                    指令队列.按优先级插入(指令)
                ELSE:
                    向 ECC 返回拒绝通知("执行队列已满")
            ELSE:
                SET internal_state = STATE_DISPATCH
                执行指令分发(指令)

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            向 ag-ecc-12 上报执行状态

        SLEEP 10ms


FUNCTION 执行指令分发(指令):
    活跃任务表[指令.指令ID] = {指令: 指令, 开始时间: NOW()}

    CASE 指令.工具类型 OF:
        "API":
            向 ag-mcc-06 发送 API 调用分发指令(指令)
        "CODE":
            向 ag-mcc-07 发送代码执行分发指令(指令)
        "FILE":
            向 ag-mcc-08 发送文件操作分发指令(指令)
        DEFAULT:
            构建失败回执(指令.指令ID, "未知工具类型: " + 指令.工具类型)
            从活跃任务表中移除(指令.指令ID)

    SET internal_state = STATE_WAIT
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 指令中工具类型为未知类型 | 立即返回失败回执，错误码=UNKNOWN_TOOL_TYPE | — |
| 并发任务数达到上限 | 将新指令加入优先级队列等待 | 有任务完成释放并发槽位 |
| 指令队列达到最大长度 | 拒绝新指令，返回"执行队列已满"通知 | 队列消化至有空位 |
| 某执行模块超时未返回结果 | 触发 ag-mcc-02 超时管理，超时后返回超时回执 | 执行模块恢复正常 |
| 收到重复指令ID | 检查活跃任务表，若已存在则返回"指令重复"拒绝 | — |
| 紧急熔断 | 清空指令队列，中断所有活跃任务，返回中断回执 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CerebellumBus | 读 | 工具调用指令 | 只读 | ag-ecc-12 下发 |
| CerebellumBus | 写 | 执行回执 | 专属写入 | 向 ag-ecc-12 返回 |
| 内部调度总线 | 写 | API调用/代码执行/文件操作分发指令 | 专属写入 | 向 ag-mcc-06/07/08 发送 |
| 内部调度总线 | 读 | 各执行模块返回的结果 | 只读 | ag-mcc-06/07/08 返回 |
| 内部调度总线 | 写 | 执行状态上报 | 周期性写入 | 向 ag-ecc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| X-01 | 本模块为 MCC 行动执行层的唯一入口，所有来自 ECC 的指令必须经本模块接收与路由，禁止执行模块直接接收外部指令 |
| X-02 | 本模块仅负责任务分发与结果汇总，不参与任何工具选择或参数决策 |
| X-03 | 所有指令必须携带 ECC-12 签发的有效安全令牌，令牌无效的指令直接拒绝 |
| X-04 | 执行超时后自动中断并上报，不得无限期等待 |
| X-05 | 紧急熔断时必须立即清空指令队列并中断所有活跃任务，不得有任务残留执行 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M01-01 | `IDLE`，收到 API 类型指令 | 工具调用指令（工具类型=API，工具=weather_api） | 分发至 ag-mcc-06，状态切换至 WAITING_RESULTS |
| TC-M01-02 | `IDLE`，收到 CODE 类型指令 | 工具调用指令（工具类型=CODE） | 分发至 ag-mcc-07 |
| TC-M01-03 | `IDLE`，收到 FILE 类型指令 | 工具调用指令（工具类型=FILE） | 分发至 ag-mcc-08 |
| TC-M01-04 | `WAITING_RESULTS`，并发数已满 | 新工具调用指令（并发数=10） | 加入优先级队列等待 |
| TC-M01-05 | `IDLE`，收到未知工具类型指令 | 工具调用指令（工具类型=UNKNOWN） | 立即返回失败回执，错误码=UNKNOWN_TOOL_TYPE |
| TC-M01-06 | `WAITING_RESULTS`，收到执行模块返回结果 | API 调用结果（状态=成功） | 汇总构建闭环回执，发送至 ag-ecc-12 |
