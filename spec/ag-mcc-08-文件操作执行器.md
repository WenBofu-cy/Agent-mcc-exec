# ag-mcc-08-文件操作执行器 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-08 |
| 模块名称 | 文件操作执行器 |
| 所属分区 | 三、调用执行引擎 |
| 核心职责 | 在用户授权的文件路径范围内，安全地执行文件的读取、写入、创建、删除、列表查询等操作。基于 ag-mcc-05（工具参数校验器）已验证的路径白名单与安全约束，对每次文件操作进行路径合法性二次校验，防止路径穿越攻击。所有操作均在限定的工作目录下执行，操作完成后记录详细的文件变更审计日志。不参与文件内容的解析或业务决策，仅负责文件 I/O 的精准执行与结果返回 |
| 依赖模块 | ag-mcc-01（执行调度核心，下发文件操作指令）、ag-mcc-04（工具注册中心，获取文件操作权限与路径白名单配置）、ag-mcc-05（工具参数校验器，预校验路径合法性） |
| 被依赖模块 | ag-mcc-01（返回文件操作执行结果）、ag-mcc-03（发送资源释放通知）、ag-mcc-12（执行日志记录单元，记录文件变更审计日志） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待指令 | `WAITING_COMMAND` | 无待执行的文件操作，等待调度核心下发 | 系统初始化完成，无活跃任务 |
| 操作执行中 | `EXECUTING` | 正在执行具体的文件 I/O 操作 | 收到文件操作指令且路径校验通过 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 文件操作指令 | Struct（指令ID + 步骤ID + 计划ID + 操作类型：读/写/创建/删除/列表/移动 + 目标路径 + 数据内容 + 编码方式 + 安全令牌） | ag-mcc-01 执行调度核心（通过内部调度总线） | ECC 需要执行文件操作时 | **高** |
| 路径白名单配置 | Struct（用户授权根目录 + 禁止的路径模式列表 + 允许的操作类型 + 最大文件大小限制 + 禁止的文件扩展名列表） | ag-mcc-04 工具注册中心 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 文件操作结果 | Struct（指令ID + 操作状态：成功/失败/权限拒绝/路径违规 + 操作详情 + 文件大小 + 耗时 + 文件哈希） | ag-mcc-01 执行调度核心（通过内部调度总线） | 操作完成或拒绝时 | **高** |
| 文件变更审计日志 | Struct（指令ID + 操作类型 + 目标路径 + 操作时间 + 操作结果 + 文件哈希 + 数据大小） | ag-mcc-12 执行日志记录单元 | 每次文件操作完成时 | 普通 |
| 资源释放通知 | Struct（指令ID + 释放的存储KB） | ag-mcc-03 资源配额管控单元 | 文件删除操作完成时 | 普通 |
| 操作状态上报 | Struct（当前状态 + 今日操作次数 + 各操作类型分布 + 成功率 + 平均耗时） | ag-mcc-12 执行日志记录单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 文件操作类型与约束

### 一、支持的操作类型

| 操作类型 | 标识 | 说明 | 路径要求 | 数据要求 |
|----------|:---:|------|------|------|
| 读取文件 | `READ` | 读取指定文件的全部或部分内容 | 路径必须在白名单内，文件必须存在 | 无 |
| 写入文件 | `WRITE` | 将数据写入指定文件（覆盖或追加） | 路径必须在白名单内 | 需要提供写入的数据内容 |
| 创建文件 | `CREATE` | 创建新文件并可选写入初始内容 | 路径必须在白名单内，文件不可已存在 | 可选初始内容 |
| 删除文件 | `DELETE` | 删除指定文件 | 路径必须在白名单内，文件必须存在 | 无 |
| 列出目录 | `LIST` | 列出指定目录下的文件与子目录 | 路径必须在白名单内，且为目录 | 无 |
| 移动/重命名 | `MOVE` | 移动或重命名文件 | 源路径与目标路径均须在白名单内 | 无 |

### 二、安全路径校验规则

| 校验项 | 规则 | 不通过时的处理 |
|----------|------|------|
| 白名单根目录 | 目标路径必须位于用户授权的根目录之下（如 `/home/user/workspace/`） | 拒绝操作，返回路径违规 |
| 路径穿越检测 | 路径中不得包含 `../` 或 `..\\` 等向上穿越特征 | 拒绝操作，返回安全告警 |
| 符号链接限制 | 禁止跟随指向白名单目录外的符号链接 | 拒绝操作，返回路径违规 |
| 禁止的文件扩展名 | 不得操作 `.exe`, `.sh`, `.bat`, `.dll` 等可执行文件 | 拒绝操作，返回文件类型限制 |
| 最大文件大小 | 单次读取/写入不得超过 100MB | 拒绝操作，返回文件大小超限 |
| 禁止的目录模式 | 不得操作系统目录（`/etc`, `/sys`, `C:\Windows` 等） | 拒绝操作，返回路径违规 |


## 核心处理逻辑

```
FUNCTION file_executor_main_loop():
    STATE_WAIT = WAITING_COMMAND
    STATE_EXEC = EXECUTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    加载路径白名单配置
    加载禁止的路径模式与文件扩展名列表

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收文件操作指令
        IF 收到 ag-mcc-01 文件操作指令:
            SET internal_state = STATE_EXEC
            指令 = 文件操作指令
            开始时间 = NOW()

            // 2a. 路径合法性二次校验
            路径校验结果 = 二次校验路径(指令.目标路径, 指令.操作类型)
            IF 路径校验结果.不通过:
                向 ag-mcc-01 返回操作结果(失败, 路径校验结果.拒绝原因)
                IF 路径校验结果.严重程度 == "安全告警":
                    向 ag-mcc-12 发送文件变更审计日志(指令, "路径违规", 路径校验结果.详情)
                SET internal_state = STATE_WAIT
                CONTINUE

            // 2b. 根据操作类型执行
            CASE 指令.操作类型 OF:
                "READ":
                    文件内容 = 安全读取文件(指令.目标路径, 指令.编码方式)
                    IF 读取成功:
                        文件哈希 = 计算文件哈希(指令.目标路径)
                        向 ag-mcc-01 返回操作结果(成功, 文件内容, 文件大小, 文件哈希)
                    ELSE:
                        向 ag-mcc-01 返回操作结果(失败, "文件读取失败")

                "WRITE":
                    IF 指令.数据内容 == None:
                        向 ag-mcc-01 返回操作结果(失败, "写入数据为空")
                        CONTINUE
                    写入结果 = 安全写入文件(指令.目标路径, 指令.数据内容, 指令.编码方式)
                    IF 写入结果.成功:
                        向 ag-mcc-01 返回操作结果(成功, 写入字节数=写入结果.字节数)
                    ELSE:
                        向 ag-mcc-01 返回操作结果(失败, 写入结果.错误原因)

                "CREATE":
                    IF 文件已存在(指令.目标路径):
                        向 ag-mcc-01 返回操作结果(失败, "文件已存在")
                        CONTINUE
                    创建结果 = 安全创建文件(指令.目标路径, 指令.数据内容)
                    向 ag-mcc-01 返回操作结果(创建结果.成功, 创建结果.详情)

                "DELETE":
                    删除结果 = 安全删除文件(指令.目标路径)
                    IF 删除结果.成功:
                        向 ag-mcc-03 发送资源释放通知(指令ID, 删除结果.释放空间KB)
                    向 ag-mcc-01 返回操作结果(删除结果.成功, 删除结果.详情)

                "LIST":
                    列表结果 = 安全列出目录(指令.目标路径)
                    向 ag-mcc-01 返回操作结果(列表结果.成功, 文件列表=列表结果.文件列表)

                "MOVE":
                    源路径校验 = 二次校验路径(指令.源路径, "MOVE")
                    目标路径校验 = 二次校验路径(指令.目标路径, "MOVE")
                    IF 源路径校验.不通过 OR 目标路径校验.不通过:
                        向 ag-mcc-01 返回操作结果(失败, "路径校验不通过")
                        CONTINUE
                    移动结果 = 安全移动文件(指令.源路径, 指令.目标路径)
                    向 ag-mcc-01 返回操作结果(移动结果.成功, 移动结果.详情)

            // 2c. 记录审计日志
            操作耗时 = NOW() - 开始时间
            向 ag-mcc-12 发送文件变更审计日志(指令, 操作结果.状态, 操作耗时)

            SET internal_state = STATE_WAIT

        // 第3步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报操作状态

        SLEEP 10ms


FUNCTION 二次校验路径(路径, 操作类型):
    白名单根目录 = 路径白名单配置.用户授权根目录

    // 检查是否在白名单根目录下
    IF NOT 路径.startswith(白名单根目录):
        RETURN {不通过: True, 拒绝原因: "路径不在授权范围内", 严重程度: "路径违规"}

    // 检查路径穿越
    IF 路径 包含 "../" OR 路径 包含 "..\\":
        RETURN {不通过: True, 拒绝原因: "检测到路径穿越", 严重程度: "安全告警"}

    // 检查禁止的文件扩展名
    文件扩展名 = 提取文件扩展名(路径)
    IF 文件扩展名 IN 禁止扩展名列表:
        RETURN {不通过: True, 拒绝原因: f"禁止操作的文件类型: {文件扩展名}", 严重程度: "文件类型限制"}

    // 检查禁止的目录模式
    FOR EACH 禁止模式 IN 禁止目录模式列表:
        IF 路径 匹配 禁止模式:
            RETURN {不通过: True, 拒绝原因: f"禁止操作的目录: {禁止模式}", 严重程度: "路径违规"}

    RETURN {不通过: False}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 目标路径指向不存在的文件（读取/删除时） | 返回失败，错误码=FILE_NOT_FOUND | — |
| 目标路径指向已存在的文件（创建时） | 返回失败，错误码=FILE_ALREADY_EXISTS | — |
| 磁盘空间不足（写入/创建时） | 返回失败，错误码=DISK_FULL | 磁盘空间释放 |
| 文件被其他进程锁定 | 重试1次（间隔500ms），仍失败则返回失败，错误码=FILE_LOCKED | 文件锁释放 |
| 读取的文件超过大小上限 | 截断返回前 100MB 内容，标记"文件过大已截断" | — |
| 移动操作时源路径与目标路径相同 | 返回失败，错误码=INVALID_OPERATION | — |
| 紧急熔断 | 完成当前正在执行的操作（保证数据一致性），暂停接收新指令 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 文件操作指令 | 只读 | ag-mcc-01 发送 |
| 内部调度总线 | 读 | 路径白名单配置 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 写 | 文件操作结果 | 专属写入 | 向 ag-mcc-01 返回 |
| 内部调度总线 | 写 | 文件变更审计日志 | 专属写入 | 向 ag-mcc-12 发送 |
| 内部调度总线 | 写 | 资源释放通知 | 专属写入 | 向 ag-mcc-03 发送 |
| 内部调度总线 | 写 | 操作状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| F-01 | 所有文件操作必须在用户授权的白名单根目录下执行，禁止访问授权范围外的任何路径 |
| F-02 | 路径穿越攻击检测为强制性安全检查，任何向上穿越的路径必须被拒绝并记录安全告警 |
| F-03 | 禁止操作可执行文件（.exe, .sh, .bat 等），防止恶意代码植入 |
| F-04 | 所有文件操作必须记录完整的审计日志，包括操作类型、路径、时间、结果与文件哈希 |
| F-05 | 文件写入操作不支持追加模式写入系统关键配置文件（如 /etc 下的任何文件） |
| F-06 | 删除操作不可逆，执行前必须在审计日志中记录文件的完整元数据（大小、哈希、修改时间） |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M08-01 | `WAITING_COMMAND`，正常读取文件 | 文件操作指令（操作类型=READ，路径=白名单内存在文件） | 返回成功，文件内容，文件哈希 |
| TC-M08-02 | `WAITING_COMMAND`，正常写入文件 | 文件操作指令（操作类型=WRITE，路径=白名单内，数据="hello"） | 写入成功，返回写入字节数 |
| TC-M08-03 | `WAITING_COMMAND`，路径穿越攻击 | 文件操作指令（路径="../../../etc/passwd"） | 拒绝操作，返回路径违规，记录安全告警 |
| TC-M08-04 | `WAITING_COMMAND`，禁止的文件扩展名 | 文件操作指令（操作类型=READ，路径="script.sh"） | 拒绝操作，返回文件类型限制 |
| TC-M08-05 | `WAITING_COMMAND`，正常删除文件 | 文件操作指令（操作类型=DELETE，路径=白名单内文件） | 删除成功，发送资源释放通知 |
| TC-M08-06 | `WAITING_COMMAND`，文件不存在 | 文件操作指令（操作类型=READ，路径=白名单内不存在的文件） | 返回失败，错误码=FILE_NOT_FOUND |
