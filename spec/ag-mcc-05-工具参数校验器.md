# ag-mcc-05-工具参数校验器 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-05 |
| 模块名称 | 工具参数校验器 |
| 所属分区 | 二、工具管理集群 |
| 核心职责 | 在工具调用指令下发至具体执行模块之前，对指令中携带的参数进行合法性校验。基于 ag-mcc-04（工具注册中心）中存储的各工具参数模板与约束规则，逐一检查每个参数的类型、格式、取值范围、依赖关系是否合规。校验通过则放行至执行模块；校验不通过则向 ag-mcc-01（执行调度核心）返回详细的参数错误报告，阻止无效调用进入执行层。同时支持为 ag-ecc-03（工具选择模块）提供参数预校验能力，辅助工具选择决策。不参与工具的实际调用，仅负责参数的合法性检查 |
| 依赖模块 | ag-mcc-04（工具注册中心，获取参数模板与约束规则） |
| 被依赖模块 | ag-mcc-01（执行调度核心，接收校验结果）、ag-ecc-03（工具选择模块，可选地调用参数预校验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待请求 | `WAITING_REQUEST` | 无待校验的指令，等待输入 | 系统初始化完成，校验队列为空 |
| 校验进行中 | `VALIDATING` | 正在对指令参数进行逐项校验 | 收到参数校验请求 |
| 校验完成 | `VALIDATED` | 校验完成，已输出结果 | 全部参数校验完毕 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 参数校验请求 | Struct（指令ID + 工具名称 + 工具类型 + 参数键值对 + 校验严格度：宽松/标准/严格 + 请求来源） | ag-mcc-01 执行调度核心（通过内部调度总线） | 每次指令分发前 | **高** |
| 参数预校验请求 | Struct（请求ID + 工具名称 + 参数键值对 + 校验严格度） | ag-ecc-03 工具选择模块（通过 MemoryBus 转发） | 工具选择过程中需要评估参数可行性时 | 普通 |
| 工具参数模板 | Struct（工具名称 + 参数模板：参数名 → {类型, 是否必填, 默认值, 取值范围, 格式正则, 依赖关系}） | ag-mcc-04 工具注册中心 | 系统初始化加载，配置更新时 | 普通 |

|--------|----------|----------|----------|:---:|
| 参数校验结果 | Struct（指令ID + 校验结果：通过/失败 + 错误列表：参数名→错误原因 + 修正建议 + 校验耗时） | ag-mcc-01 执行调度核心 | 校验完成时 | **高** |

## 参数校验规则

### 一、校验严格度

| 严格度 | 标识 | 适用场景 | 校验行为 |
|:---:|:---:|------|------|
| 宽松 | `LOOSE` | 工具选择阶段的预校验 | 仅校验必填参数是否存在，不检查格式与取值 |
| 标准 | `STANDARD` | 一般工具调用 | 校验必填参数、类型匹配、格式正则、取值范围 |
| 严格 | `STRICT` | 安全敏感的工具调用 | 在标准基础上增加参数间依赖关系校验、业务逻辑一致性检查 |

### 二、校验项清单

| 校验项 | 说明 | 不通过时的错误码 | 适用严格度 |
|----------|------|:---:|:---:|
| 必填参数检查 | 检查模板中标记为 required 的参数是否全部提供 | MISSING_PARAM | 宽松/标准/严格 |
| 类型匹配检查 | 检查参数值的类型是否与模板定义一致（string/int/float/bool/array/object） | TYPE_MISMATCH | 标准/严格 |
| 格式正则检查 | 检查字符串参数是否符合模板定义的格式正则（如邮箱、URL、日期） | FORMAT_ERROR | 标准/严格 |
| 取值范围检查 | 检查数值参数是否在模板定义的 min/max 范围内，枚举参数是否在允许值列表中 | RANGE_ERROR | 标准/严格 |
| 依赖关系检查 | 检查条件依赖参数——当参数A取某值时，参数B是否按要求存在 | DEPENDENCY_ERROR | 严格 |
| 未知参数检查 | 检查是否传入了模板未定义的参数（根据工具配置决定是否拒绝） | UNKNOWN_PARAM | 严格 |

### 三、安全敏感参数校验

| 参数特征 | 额外校验 |
|----------|------|
| 文件路径参数 | 校验路径是否在白名单目录下，禁止绝对路径穿越（如 `../../etc/passwd`） |
| 命令执行参数 | 校验是否包含命令注入特征（`;`, `|`, `&&`, `$()` 等） |
| URL 参数 | 校验协议是否为 https，域名是否在白名单中 |
| SQL 参数 | 校验是否包含 SQL 注入特征（`DROP`, `UNION`, `--` 等） |


## 核心处理逻辑

```
FUNCTION param_validator_main_loop():
    STATE_WAIT = WAITING_REQUEST
    STATE_VALIDATE = VALIDATING
    STATE_DONE = VALIDATED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    加载所有工具的参数模板与约束规则
    初始化校验统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收校验请求
        IF 收到参数校验请求 OR 参数预校验请求:
            SET internal_state = STATE_VALIDATE
            请求 = 校验请求

            // 2a. 获取该工具的参数模板
            参数模板 = 向 ag-mcc-04 查询工具参数模板(请求.工具名称)
            IF 参数模板 == None:
                向请求来源返回校验结果(失败, "工具未注册或参数模板不存在")
                SET internal_state = STATE_WAIT
                CONTINUE

            // 2b. 逐项校验
            错误列表 = []

            // 必填参数检查
            FOR EACH 参数定义 IN 参数模板:
                IF 参数定义.是否必填 AND 参数定义.参数名 NOT IN 请求.参数键值对:
                    错误列表.append({参数名: 参数定义.参数名, 错误: "MISSING_PARAM", 原因: "必填参数缺失"})

            // 类型匹配检查（标准/严格模式）
            IF 请求.校验严格度 != "宽松":
                FOR EACH 参数名, 参数值 IN 请求.参数键值对:
                    IF 参数名 IN 参数模板:
                        期望类型 = 参数模板[参数名].类型
                        IF NOT 类型匹配(参数值, 期望类型):
                            错误列表.append({参数名: 参数名, 错误: "TYPE_MISMATCH", 原因: f"期望{期望类型}, 实际{实际类型}"})

            // 取值范围检查（标准/严格模式）
            IF 请求.校验严格度 != "宽松":
                FOR EACH 参数名, 参数值 IN 请求.参数键值对:
                    IF 参数名 IN 参数模板 AND 参数模板[参数名].取值范围 != None:
                        IF NOT 在取值范围内(参数值, 参数模板[参数名].取值范围):
                            错误列表.append({参数名: 参数名, 错误: "RANGE_ERROR", 原因: f"超出允许范围"})

            // 安全敏感参数检查（标准/严格模式）
            IF 请求.校验严格度 != "宽松":
                FOR EACH 参数名, 参数值 IN 请求.参数键值对:
                    IF 参数模板[参数名].安全敏感:
                        安全检查结果 = 执行安全敏感检查(参数名, 参数值)
                        IF 安全检查结果.不通过:
                            错误列表.append({参数名: 参数名, 错误: "SECURITY_ERROR", 原因: 安全检查结果.原因})

            // 2c. 输出校验结果
            SET internal_state = STATE_DONE

            IF 错误列表为空:
                向请求来源返回校验结果(通过, 请求.指令ID)
            ELSE:
                修正建议 = 生成修正建议(错误列表, 参数模板)
                向请求来源返回校验结果(失败, 请求.指令ID, 错误列表, 修正建议)

            更新校验统计计数器
            SET internal_state = STATE_WAIT

        // 第3步：周期性统计上报
        IF 距上次统计上报 >= 60秒:
            向 ag-mcc-12 上报校验统计

        SLEEP 10ms


FUNCTION 执行安全敏感检查(参数名, 参数值):
    IF 参数名 包含 "path" OR 参数名 包含 "file":
        IF 参数值 包含 "../" OR 参数值 包含 "..\\":
            RETURN {不通过: True, 原因: "检测到路径穿越攻击"}
    IF 参数名 包含 "command" OR 参数名 包含 "cmd":
        IF 参数值 包含 ";" OR 参数值 包含 "|" OR 参数值 包含 "&&":
            RETURN {不通过: True, 原因: "检测到命令注入特征"}
    IF 参数名 包含 "url":
        IF NOT 参数值.startswith("https://"):
            RETURN {不通过: True, 原因: "仅允许HTTPS协议的URL"}
    RETURN {不通过: False}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 工具参数模板不存在（工具未注册） | 直接拒绝，返回"工具未注册或参数模板不存在" | 工具被注册到 ag-mcc-04 |
| 校验请求中参数键值对为空 | 仍进行必填参数检查——若模板中所有参数均为可选，则通过；否则返回缺失参数列表 | — |
| 参数值类型无法识别 | 保守处理：标记为 TYPE_MISMATCH，建议修正 | — |
| 安全敏感检查发现攻击特征 | 立即拒绝，记录安全事件日志，通知 ag-ecc-04 安全仲裁模块 | — |
| 紧急熔断 | 暂停校验，未完成的请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 参数校验请求 | 只读 | ag-mcc-01 发送 |
| MemoryBus | 读 | 参数预校验请求 | 只读 | ag-ecc-03 发送（通过 ag-ecc-12 转发） |
| 内部调度总线 | 读 | 工具参数模板 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 写 | 参数校验结果 | 专属写入 | 向 ag-mcc-01 返回 |
| MemoryBus | 写 | 参数预校验结果 | 专属写入 | 向 ag-ecc-03 返回 |
| 内部调度总线 | 写 | 校验统计上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| V-01 | 本模块仅校验参数合法性，不缓存、不存储、不转发任何工具参数的实际值 |
| V-02 | 安全敏感参数检查必须在标准及以上严格度强制执行，不得跳过 |
| V-03 | 检测到攻击特征时，必须立即拒绝并记录安全事件日志，不得仅返回警告 |
| V-04 | 参数校验结果中的错误详情不得泄露工具模板的内部实现细节 |
| V-05 | 参数预校验仅检查格式合法性，不得向请求方返回工具的完整参数模板 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M05-01 | `WAITING_REQUEST`，参数完全合规 | 校验请求（工具=weather_api，参数={city:"北京"}，模板中 city 为必填 string） | 校验通过，返回成功 |
| TC-M05-02 | `WAITING_REQUEST`，缺少必填参数 | 校验请求（工具=weather_api，参数={}，模板中 city 为必填） | 校验失败，错误列表含 MISSING_PARAM |
| TC-M05-03 | `WAITING_REQUEST`，参数类型不匹配 | 校验请求（工具=weather_api，参数={city:123}，模板中 city 为 string） | 校验失败，错误列表含 TYPE_MISMATCH |
| TC-M05-04 | `WAITING_REQUEST`，取值范围超限 | 校验请求（工具=search，参数={page:0}，模板中 page 取值 min=1） | 校验失败，错误列表含 RANGE_ERROR |
| TC-M05-05 | `WAITING_REQUEST`，检测到路径穿越 | 校验请求（工具=file_read，参数={path:"../../../etc/passwd"}） | 校验失败，错误列表含 SECURITY_ERROR |
| TC-M05-06 | `WAITING_REQUEST`，工具未注册 | 校验请求（工具=unknown_tool） | 校验失败，返回"工具未注册或参数模板不存在" |
