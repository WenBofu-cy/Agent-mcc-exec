# ag-mcc-11-闭环反馈单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-11 |
| 模块名称 | 闭环反馈单元 |
| 所属分区 | 四、反馈与日志 |
| 核心职责 | 作为 MCC 行动执行层的结果汇总与反馈中枢，接收各执行模块（API调用引擎、代码执行沙箱、文件操作执行器）返回并经 ag-mcc-09（结果校验器）校验、ag-mcc-10（执行偏差监控）偏差分析后的执行结果。将所有结果统一封装为标准化闭环回执，通过 CerebellumBus 上报至 ag-ecc-12（资源调度模块），完成从指令下发到结果回传的完整执行闭环。同时汇总执行过程中的异常事件、超时通知与偏差告警，一并上报供 ECC 认知大脑进行任务恢复与决策优化。不参与结果校验或偏差分析，仅负责结果的最终汇总、封装与回传 |
| 依赖模块 | ag-mcc-09（结果校验器，提供校验后执行结果）、ag-mcc-10（执行偏差监控，提供偏差分析报告与告警）、ag-mcc-02（工具超时管理器，提供超时事件日志） |
| 被依赖模块 | ag-ecc-12（资源调度模块，接收闭环回执）、ag-mcc-12（执行日志记录单元，记录闭环事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待结果 | `WAITING_RESULT` | 无待汇总的执行结果 | 系统初始化完成，等待各模块输入 |
| 汇总封装中 | `AGGREGATING` | 正在将校验后结果与偏差分析报告封装为闭环回执 | 收到校验后执行结果和偏差分析报告 |
| 回传中 | `REPORTING` | 正在通过 CerebellumBus 上报闭环回执 | 闭环回执封装完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 校验后执行结果 | Struct（指令ID + 步骤ID + 计划ID + 工具名称 + 执行状态 + 校验标记 + 清洗后输出数据 + 实际耗时 + 资源消耗） | ag-mcc-09 结果校验器（通过内部调度总线） | 每次结果校验完成时 | **高** |
| 偏差分析报告 | Struct（指令ID + 步骤ID + 计划ID + 综合偏差度 + 各维度偏差详情 + 告警等级 + 建议处理） | ag-mcc-10 执行偏差监控（通过内部调度总线） | 每次偏差分析完成时 | **高** |
| 超时事件日志 | Struct（指令ID + 工具名称 + 超时阈值 + 实际耗时 + 时间戳） | ag-mcc-02 工具超时管理器（通过内部调度总线） | 每次检测到工具调用超时时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 闭环回执 | Struct（指令ID + 步骤ID + 计划ID + 工具名称 + 执行状态 + 输出数据 + 校验标记 + 偏差分析摘要 + 超时标记 + 资源消耗汇总 + 时间戳） | ag-ecc-12 资源调度模块（通过 CerebellumBus） | 所有相关输入收集完成时 | **高** |
| 闭环事件日志 | Struct（指令ID + 闭环状态 + 各环节耗时 + 异常标记 + 时间戳） | ag-mcc-12 执行日志记录单元 | 每次闭环回执发送完成时 | 普通 |
| 反馈状态上报 | Struct（当前状态 + 今日闭环数 + 平均闭环耗时 + 异常闭环率） | ag-mcc-12 执行日志记录单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 闭环回执结构定义

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| 指令ID | String | 原始工具调用指令的唯一标识 | ag-mcc-01 |
| 步骤ID | String | 关联的任务步骤标识 | ag-mcc-01 |
| 计划ID | String | 关联的任务计划标识 | ag-mcc-01 |
| 工具名称 | String | 被调用的工具名称 | ag-mcc-01 |
| 执行状态 | Enum | 成功/失败/超时/异常 | 各执行模块 |
| 输出数据 | Struct | 清洗后的执行输出数据 | ag-mcc-09 |
| 校验标记 | String | 通过/格式异常/字段缺失/数据截断/安全告警 | ag-mcc-09 |
| 偏差分析摘要 | Struct | 综合偏差度 + 告警等级 + 关键偏差维度 | ag-mcc-10 |
| 超时标记 | Boolean | 是否触发过超时 | ag-mcc-02 |
| 实际耗时 | Float | 总执行耗时（秒） | 各执行模块 |
| 资源消耗汇总 | Struct | API次数/内存MB/Token/存储KB | ag-mcc-03 |
| 时间戳 | Timestamp | 闭环回执生成时间 | 本模块 |


## 核心处理逻辑

```
FUNCTION feedback_loop_main_loop():
    STATE_WAIT = WAITING_RESULT
    STATE_AGGREGATE = AGGREGATING
    STATE_REPORT = REPORTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化待汇总结果表 = {}  // 指令ID → {校验结果, 偏差报告, 超时日志}
    初始化闭环统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            FOR EACH 未完成的闭环 IN 待汇总结果表:
                强制构建部分闭环回执并上报(未完成的闭环)
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：收集校验后执行结果
        IF 收到 ag-mcc-09 校验后执行结果:
            IF 结果.指令ID NOT IN 待汇总结果表:
                待汇总结果表[结果.指令ID] = {}
            待汇总结果表[结果.指令ID]["校验结果"] = 结果
            检查是否收集完成(结果.指令ID)

        // 第3步：收集偏差分析报告
        IF 收到 ag-mcc-10 偏差分析报告:
            IF 报告.指令ID NOT IN 待汇总结果表:
                待汇总结果表[报告.指令ID] = {}
            待汇总结果表[报告.指令ID]["偏差报告"] = 报告
            检查是否收集完成(报告.指令ID)

        // 第4步：收集超时事件日志
        IF 收到 ag-mcc-02 超时事件日志:
            IF 日志.指令ID NOT IN 待汇总结果表:
                待汇总结果表[日志.指令ID] = {}
            待汇总结果表[日志.指令ID]["超时日志"] = 日志
            检查是否收集完成(日志.指令ID)

        // 第5步：收集完成，封装闭环回执
        IF 检查是否收集完成(指令ID):
            SET internal_state = STATE_AGGREGATE
            汇总数据 = 待汇总结果表[指令ID]
            校验结果 = 汇总数据.GET("校验结果")
            偏差报告 = 汇总数据.GET("偏差报告")
            超时日志 = 汇总数据.GET("超时日志")

            // 封装闭环回执
            闭环回执 = 构建闭环回执(
                指令ID=指令ID,
                步骤ID=校验结果.步骤ID,
                计划ID=校验结果.计划ID,
                工具名称=校验结果.工具名称,
                执行状态=校验结果.执行状态,
                输出数据=校验结果.清洗后输出数据,
                校验标记=校验结果.校验标记,
                偏差分析摘要={
                    综合偏差度: 偏差报告.综合偏差度 IF 偏差报告 != None ELSE 0,
                    告警等级: 偏差报告.告警等级 IF 偏差报告 != None ELSE "正常",
                    关键偏差维度: 偏差报告.偏差维度 IF 偏差报告 != None ELSE []
                },
                超时标记=超时日志 != None,
                实际耗时=校验结果.实际耗时,
                资源消耗汇总=校验结果.资源消耗,
                时间戳=NOW()
            )

            // 发送闭环回执至 ECC
            SET internal_state = STATE_REPORT
            向 ag-ecc-12 发送闭环回执(闭环回执)

            // 记录闭环事件日志
            向 ag-mcc-12 发送闭环事件日志(
                指令ID=指令ID,
                闭环状态="完成",
                异常标记=IF 偏差报告.告警等级 != "正常" OR 超时日志 != None THEN True ELSE False
            )

            // 从待汇总表中移除
            从待汇总结果表中移除(指令ID)

            // 更新统计
            更新闭环统计()

            SET internal_state = STATE_WAIT

        // 第6步：超时未收集完成的处理
        FOR EACH 指令ID, 汇总数据 IN 待汇总结果表:
            IF 汇总数据中存在"校验结果" AND (NOW() - 校验结果.时间戳) > 10秒:
                // 超时未收集到偏差报告，使用默认值构建闭环回执
                强制构建部分闭环回执并上报(指令ID, 汇总数据)
                从待汇总结果表中移除(指令ID)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报反馈状态

        SLEEP 10ms


FUNCTION 检查是否收集完成(指令ID):
    IF 指令ID NOT IN 待汇总结果表:
        RETURN False
    汇总数据 = 待汇总结果表[指令ID]

    // 必须至少有校验结果
    IF "校验结果" NOT IN 汇总数据:
        RETURN False

    // 偏差报告和超时日志是可选的
    // 如果执行状态为成功，且校验标记为"通过"，即使没有偏差报告也可闭环
    校验结果 = 汇总数据["校验结果"]
    IF 校验结果.执行状态 == "成功" AND 校验结果.校验标记 == "通过":
        RETURN True

    // 否则需要偏差报告
    RETURN "偏差报告" IN 汇总数据
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 某指令的偏差报告超过10秒未到达 | 使用默认偏差值（综合偏差度=0，告警等级=正常）强制构建闭环回执 | — |
| 闭环回执发送失败（CerebellumBus 不可用） | 缓存待发送的回执，定时重试（最多3次） | CerebellumBus 恢复 |
| 同一指令收到重复的校验结果 | 以最新收到的为准，覆盖旧数据 | — |
| 执行状态为"失败"但无偏差报告 | 自动生成偏差报告（综合偏差度=1.0，告警等级=严重） | — |
| 紧急熔断 | 立即强制构建所有未完成的闭环回执并发送，清空待汇总表 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 校验后执行结果 | 只读 | ag-mcc-09 发送 |
| 内部调度总线 | 读 | 偏差分析报告 | 只读 | ag-mcc-10 发送 |
| 内部调度总线 | 读 | 超时事件日志 | 只读 | ag-mcc-02 发送 |
| CerebellumBus | 写 | 闭环回执 | 专属写入 | 向 ag-ecc-12 发送 |
| 内部调度总线 | 写 | 闭环事件日志 | 专属写入 | 向 ag-mcc-12 发送 |
| 内部调度总线 | 写 | 反馈状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| F-01 | 闭环回执中不得包含任何认证凭证、密钥或敏感用户数据，所有输出数据必须已经过 ag-mcc-09 的脱敏处理 |
| F-02 | 闭环回执发送必须加密传输（CerebellumBus 默认加密），防止中间人窃听 |
| F-03 | 未能在规定时间内完成闭环的指令，不得无限期等待，超时后必须强制构建回执上报 |
| F-04 | 闭环回执中的偏差分析摘要不得包含原始输出数据，仅保留统计级别的偏差指标 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M11-01 | `WAITING_RESULT`，全部数据按时到达 | 校验结果+偏差报告（正常） | 构建闭环回执，发送至 ag-ecc-12，标记为完成 |
| TC-M11-02 | `WAITING_RESULT`，成功结果无需偏差报告 | 校验结果（执行状态=成功，校验标记=通过） | 直接构建闭环回执，偏差摘要使用默认值 |
| TC-M11-03 | `WAITING_RESULT`，偏差报告超时未到达 | 校验结果（执行状态=成功，到达后10秒偏差报告未到） | 强制构建闭环回执，偏差摘要使用默认值 |
| TC-M11-04 | `WAITING_RESULT`，执行失败自动生成偏差 | 校验结果（执行状态=失败，校验标记=FORMAT_ERROR） | 自动生成偏差报告（综合偏差度=1.0，告警=严重），构建闭环回执 |
| TC-M11-05 | `WAITING_RESULT`，收到超时事件日志 | 校验结果+超时日志 | 闭环回执中标记超时=True |
| TC-M11-06 | `SYSTEM_PAUSED`，有待汇总数据 | 待汇总表中有未完成的数据 | 强制构建部分闭环回执并发送，清空待汇总表 |
