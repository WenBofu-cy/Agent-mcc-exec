# ag-mcc-04-工具注册中心 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-04 |
| 模块名称 | 工具注册中心 |
| 所属分区 | 二、工具管理集群 |
| 核心职责 | 作为 MCC 行动执行层的工具元数据中心，维护所有可用工具的全生命周期注册信息。管理工具名称、类型、参数模板、调用方式、配额预设、安全等级等元数据。支持运行时动态注册新工具、注销已废弃工具、更新工具版本与参数约束。为 ag-mcc-01（执行调度核心）提供工具类型路由依据，为 ag-mcc-03（资源配额管控单元）提供各工具的预设资源配额，为 ag-mcc-05（工具参数校验器）提供参数合法性校验模板。不参与工具的实际调用，仅负责工具元数据的存储、检索与版本管理 |
| 依赖模块 | 无（作为工具元数据的基础设施，不依赖其他 MCC 内部模块） |
| 被依赖模块 | ag-mcc-01（执行调度核心，查询工具类型与路由信息）、ag-mcc-03（资源配额管控单元，获取各工具资源配额预设）、ag-mcc-05（工具参数校验器，获取参数模板与约束规则）、ag-mcc-06（API调用引擎，获取 API 端点与认证配置） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 工具注册信息完整，可正常响应查询与注册 | 系统初始化完成，元数据索引构建成功 |
| 注册更新中 | `REGISTERING` | 正在注册新工具或更新已有工具信息 | 收到工具注册/更新请求 |
| 工具注销中 | `UNREGISTERING` | 正在注销指定工具，将其标记为不可用 | 收到工具注销请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 工具信息查询请求 | Struct（请求来源模块 + 查询类型：单个工具/全量/按类型筛选 + 工具名称/类型） | ag-mcc-01/03/05/06 等模块 | 需要获取工具元数据时 | **高** |
| 工具注册请求 | Struct（工具名称 + 工具类型 + 版本号 + 参数模板 + 调用端点 + 配额预设 + 安全等级 + 注册来源） | 系统管理接口或 OTA 升级系统 | 新工具上线或版本更新时 | **高** |
| 工具注销请求 | Struct（工具名称 + 注销原因 + 是否强制注销 + 授权令牌） | 系统管理接口 | 工具废弃或安全风险时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 工具信息查询结果 | Struct（工具名称 + 工具类型 + 参数模板 + 调用方式 + 配额预设 + 安全等级 + 版本号 + 当前状态） | 请求来源模块 | 查询完成时 | **高** |
| 工具注册确认 | Struct（工具名称 + 注册状态 + 新版本号 + 生效时间） | 注册请求方 | 注册/更新完成时 | **高** |
| 工具注销确认 | Struct（工具名称 + 注销状态 + 残留任务处理方式） | 注销请求方 | 注销完成时 | **高** |
| 工具变更通知 | Struct（变更类型：注册/更新/注销 + 工具名称 + 版本号 + 变更时间） | ag-mcc-01（广播至各执行模块） | 工具元数据变更时 | 普通 |


## 工具注册信息模板

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 工具名称 | String | 全局唯一标识 | weather_api |
| 工具类型 | Enum | API / CODE / FILE / LLM / DB | API |
| 版本号 | String | 语义化版本 | 2.1.0 |
| 参数模板 | JSON Schema | 参数名称、类型、是否必填、默认值、取值范围 | {"city": {"type": "string", "required": true}} |
| 调用端点 | String | API 类型工具的 URL，或代码执行入口 | https://api.weather.com/v2 |
| 认证方式 | Enum | NONE / API_KEY / OAUTH2 / TOKEN | API_KEY |
| 配额预设 | Struct | 单次调用的资源消耗预设 | {api_calls: 1, memory_mb: 10, tokens: 50} |
| 安全等级 | Enum | LOW / MEDIUM / HIGH / CRITICAL | LOW |
| 默认超时 | Integer | 秒 | 30 |
| 最大并发数 | Integer | 该工具允许的最大并行调用数 | 5 |
| 状态 | Enum | ACTIVE / DEPRECATED / DISABLED | ACTIVE |
| 注册时间 | Timestamp | UTC | 2026-06-03T10:00:00Z |
| 最后更新时间 | Timestamp | UTC | 2026-06-03T12:00:00Z |


## 核心处理逻辑

```
FUNCTION tool_registry_main_loop():
    STATE_NORMAL = NORMAL_SERVICE
    STATE_REG = REGISTERING
    STATE_UNREG = UNREGISTERING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化工具元数据库 = {}
    构建工具索引（按名称、类型、安全等级）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：处理工具信息查询
        IF 收到工具信息查询请求:
            IF 请求.查询类型 == "单个工具":
                结果 = 工具元数据库.查询(请求.工具名称)
            ELSE IF 请求.查询类型 == "全量":
                结果 = 工具元数据库.全部工具()
            ELSE IF 请求.查询类型 == "按类型筛选":
                结果 = 工具索引.按类型筛选(请求.工具类型)

            向请求来源返回工具信息查询结果(结果)

        // 第3步：处理工具注册/更新
        IF 收到工具注册请求:
            SET internal_state = STATE_REG

            // 校验必填字段
            IF 请求.工具名称为空 OR 请求.工具类型为空:
                向请求方返回注册确认(失败, "缺少必填字段")
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 检查是否已存在
            IF 请求.工具名称 IN 工具元数据库:
                旧版本 = 工具元数据库[请求.工具名称].版本号
                更新工具元数据(请求)
                新版本 = 请求.版本号
                向请求方返回注册确认(成功, "更新", 旧版本, 新版本)
            ELSE:
                创建新工具元数据条目(请求)
                向请求方返回注册确认(成功, "新增", "", 请求.版本号)

            // 发送工具变更通知
            向 ag-mcc-01 发送工具变更通知("注册/更新", 请求.工具名称)

            SET internal_state = STATE_NORMAL

        // 第4步：处理工具注销
        IF 收到工具注销请求:
            SET internal_state = STATE_UNREG

            IF 请求.工具名称 NOT IN 工具元数据库:
                向请求方返回注销确认(失败, "工具不存在")
                SET internal_state = STATE_NORMAL
                CONTINUE

            IF 请求.强制注销:
                工具元数据库[请求.工具名称].状态 = DISABLED
            ELSE:
                工具元数据库[请求.工具名称].状态 = DEPRECATED

            向请求方返回注销确认(成功, 请求.工具名称, 残留任务处理方式)

            // 发送工具变更通知
            向 ag-mcc-01 发送工具变更通知("注销", 请求.工具名称)

            SET internal_state = STATE_NORMAL

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 注册请求中工具名称与已有工具冲突 | 视为更新操作，覆盖旧版本的元数据，保留历史版本记录 | — |
| 注册请求中版本号低于或等于当前版本 | 拒绝更新，返回"版本号必须高于当前版本" | 提供更高版本号 |
| 查询请求指定的工具名称不存在 | 返回空结果 + 提示"工具未注册" | 工具被注册 |
| 注销请求指定了仍有活跃任务的工具 | 默认拒绝注销，除非请求中标记为"强制注销" | 活跃任务完成或强制注销 |
| 紧急熔断 | 暂停所有注册/注销操作，保持当前工具元数据不变 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 工具信息查询请求 | 只读 | ag-mcc-01/03/05/06 发送 |
| 管理接口 | 读 | 工具注册/注销请求 | 只读（需令牌） | 系统管理接口或 OTA 发送 |
| 内部调度总线 | 写 | 工具信息查询结果 | 专属写入 | 向请求来源返回 |
| 管理接口 | 写 | 工具注册/注销确认 | 专属写入 | 向请求方返回 |
| 内部调度总线 | 写 | 工具变更通知 | 广播 | 向 ag-mcc-01 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| R-01 | 工具注册/注销操作必须持有有效的授权令牌，防止未授权的工具注入或移除 |
| R-02 | 安全等级为 CRITICAL 的工具不可通过运行时注册接口动态注册，仅允许预置或系统级更新 |
| R-03 | 工具元数据中不得存储任何认证密钥或敏感凭证，认证方式仅记录类型 |
| R-04 | 工具注销后，其元数据不得立即删除，应保留至少 30 天供审计追溯 |
| R-05 | 工具版本更新时必须保留至少上一个版本的元数据快照，支持紧急回滚 |

| TC-M04-04 | `NORMAL_SERVICE`，版本号不升反降 | 注册请求（工具名称=weather_api，版本号=0.9.0，当前版本=1.0.0） | 拒绝更新，返回"版本号必须高于当前版本" |

| TC-M04-06 | `SYSTEM_PAUSED`，收到查询请求 | 查询请求 | 拒绝处理，保持暂停状态 |
