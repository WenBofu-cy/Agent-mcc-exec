# ag-mcc-07-代码执行沙箱 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-07 |
| 模块名称 | 代码执行沙箱 |
| 所属分区 | 三、调用执行引擎 |
| 核心职责 | 在隔离的安全环境中执行由 ECC 认知大脑下发或用户提供的代码片段。基于沙箱技术限制代码对系统资源（文件系统、网络、进程）的访问，控制执行超时与内存上限。捕获代码的标准输出、标准错误及返回值，结构化后返回至 ag-mcc-01（执行调度核心）。同时支持多种编程语言的执行运行时。不参与代码内容的审查或修改，仅负责代码的安全执行与结果捕获 |
| 依赖模块 | ag-mcc-01（执行调度核心，下发代码执行指令）、ag-mcc-04（工具注册中心，获取代码执行环境配置与语言运行时信息）、ag-mcc-03（资源配额管控单元，接收内存/CPU配额约束） |
| 被依赖模块 | ag-mcc-01（返回代码执行结果）、ag-mcc-03（发送资源释放通知） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待指令 | `WAITING_COMMAND` | 无待执行的代码任务，等待调度核心下发 | 系统初始化完成，无活跃沙箱实例 |
| 沙箱准备 | `SANDBOX_SETUP` | 正在为即将执行的代码创建隔离沙箱环境 | 收到代码执行指令 |
| 代码执行中 | `EXECUTING` | 代码在沙箱中运行 | 沙箱环境就绪，代码开始执行 |
| 结果收集中 | `COLLECTING_RESULT` | 代码执行完毕，正在收集输出与资源使用数据 | 沙箱中代码进程终止 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 代码执行指令 | Struct（指令ID + 步骤ID + 计划ID + 语言类型 + 源代码 + 标准输入数据 + 超时时间 + 内存上限MB + CPU上限 + 安全令牌 + 环境变量列表） | ag-mcc-01 执行调度核心（通过内部调度总线） | ECC 需要执行代码片段时 | **高** |
| 语言运行时配置 | Struct（语言名称 + 沙箱镜像版本 + 预装库清单 + 可用系统调用白名单 + 文件系统挂载点） | ag-mcc-04 工具注册中心 | 每次接收到代码执行指令时查询 | **高** |
| 资源约束参数 | Struct（指令ID + 最大内存MB + 最大CPU核心数 + 最大执行时间秒 + 最大输出大小KB） | ag-mcc-03 资源配额管控单元 | 随指令附带或配额校验时下发 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 代码执行结果 | Struct（指令ID + 执行状态：成功/失败/超时/内存溢出/安全违规 + 标准输出 + 标准错误 + 返回值 + 实际耗时 + 实际内存峰值MB + 退出码） | ag-mcc-01 执行调度核心（通过内部调度总线） | 代码执行完成或异常终止时 | **高** |
| 资源释放通知 | Struct（指令ID + 释放的内存MB + 释放的CPU时间） | ag-mcc-03 资源配额管控单元 | 沙箱实例销毁时 | 普通 |
| 安全违规告警 | Struct（指令ID + 违规类型 + 违规详情 + 严重等级） | ag-mcc-01（转至 ag-ecc-04 安全仲裁模块） | 检测到沙箱逃逸尝试或禁止的系统调用时 | **高** |
| 执行状态上报 | Struct（当前状态 + 活跃沙箱数 + 今日执行次数 + 成功率 + 平均执行耗时） | ag-mcc-12 执行日志记录单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 沙箱隔离策略

### 一、资源限制

| 限制项 | 默认值 | 可配置范围 | 超出时行为 |
|--------|:---:|:---:|------|
| 最大执行时间 | 30 秒 | 5 – 120 秒 | 强制终止，返回超时状态 |
| 最大内存使用 | 128 MB | 32 – 512 MB | 进程被 OOM Killer 终止，返回内存溢出 |
| 最大 CPU 核心数 | 1 核 | 1 – 2 核 | 仅分配指定核心数 |
| 最大输出大小 | 1 MB | 100 KB – 10 MB | 截断输出，标记输出截断 |
| 最大进程数 | 5 | 1 – 10 | 禁止 fork 新进程 |

### 二、安全能力限制

| 能力 | 默认策略 | 说明 |
|------|:---:|------|
| 网络访问 | **完全禁止** | 沙箱内不分配网络接口 |
| 文件系统写 | **禁止** | 除临时目录外，禁止任何写操作 |
| 文件系统读 | **白名单** | 仅允许读取预置库文件与指定的输入数据文件 |
| 系统调用 | **白名单** | 仅允许安全系统调用列表中的调用 |
| 进程间通信 | **禁止** | 禁止与沙箱外进程通信 |
| 特权提升 | **禁止** | 禁止 setuid、capabilities 等操作 |

### 三、多语言运行时

| 语言 | 默认版本 | 沙箱镜像 | 预置库 |
|------|:---:|------|------|
| Python | 3.11 | python:3.11-slim | math, json, datetime, re, collections, itertools |
| JavaScript | Node.js 20 | node:20-alpine | 无第三方库，仅 Node.js 内置模块 |
| Bash | 5.2 | alpine:3.19 | 标准 GNU 工具集（受限） |
| SQL | SQLite 3.42 | sqlite:3.42 | 仅 SQLite 内置函数 |


## 核心处理逻辑

```
FUNCTION sandbox_main_loop():
    STATE_WAIT = WAITING_COMMAND
    STATE_SETUP = SANDBOX_SETUP
    STATE_EXEC = EXECUTING
    STATE_COLLECT = COLLECTING_RESULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化活跃沙箱表 = {}  // 指令ID → 沙箱实例上下文

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            FOR EACH 活跃沙箱 IN 活跃沙箱表:
                强制终止沙箱(活跃沙箱)
                清空活跃沙箱表
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收代码执行指令
        IF 收到 ag-mcc-01 代码执行指令:
            SET internal_state = STATE_SETUP
            指令 = 代码执行指令

            // 2a. 获取运行时配置
            运行时配置 = 向 ag-mcc-04 查询语言运行时配置(指令.语言类型)
            IF 运行时配置 == None:
                向 ag-mcc-01 返回执行结果(失败, "不支持的语言类型: " + 指令.语言类型)
                SET internal_state = STATE_WAIT
                CONTINUE

            // 2b. 获取资源约束
            资源约束 = 指令.附带资源约束
            IF 资源约束 == None:
                资源约束 = 默认资源约束

            // 2c. 创建沙箱实例
            沙箱实例 = 创建沙箱(
                镜像=运行时配置.沙箱镜像,
                内存上限=资源约束.最大内存MB,
                CPU上限=资源约束.最大CPU核心数,
                超时=指令.超时时间,
                文件系统白名单=运行时配置.文件系统挂载点
            )
            活跃沙箱表[指令.指令ID] = {
                指令: 指令,
                沙箱实例: 沙箱实例,
                开始时间: NOW()
            }

            // 2d. 注入代码与输入数据
            沙箱实例.写入代码文件(指令.源代码)
            IF 指令.标准输入数据 != None:
                沙箱实例.写入标准输入(指令.标准输入数据)

            // 2e. 启动执行
            SET internal_state = STATE_EXEC
            沙箱实例.启动执行()

        // 第3步：监控沙箱执行状态
        IF internal_state == STATE_EXEC:
            FOR EACH 指令ID, 上下文 IN 活跃沙箱表:
                沙箱状态 = 上下文.沙箱实例.检查状态()

                IF 沙箱状态 == "已终止":
                    SET internal_state = STATE_COLLECT
                    终止原因 = 上下文.沙箱实例.终止原因

                    IF 终止原因 == "正常退出":
                        输出 = 上下文.沙箱实例.收集标准输出()
                        错误 = 上下文.沙箱实例.收集标准错误()
                        返回值 = 上下文.沙箱实例.获取返回值()
                        资源用量 = 上下文.沙箱实例.获取资源用量()
                        向 ag-mcc-01 返回执行结果(成功, 输出, 错误, 返回值, 资源用量)

                    ELSE IF 终止原因 == "超时":
                        向 ag-mcc-01 返回执行结果(超时, 实际耗时=指令.超时时间)

                    ELSE IF 终止原因 == "内存溢出":
                        向 ag-mcc-01 返回执行结果(内存溢出, 实际内存峰值=资源约束.最大内存MB)

                    ELSE IF 终止原因 == "安全违规":
                        违规详情 = 上下文.沙箱实例.安全审计日志()
                        向 ag-mcc-01 发送安全违规告警(指令ID, 违规详情)
                        向 ag-mcc-01 返回执行结果(安全违规, 违规详情)

                    ELSE:
                        向 ag-mcc-01 返回执行结果(失败, 终止原因)

                    // 释放资源
                    向 ag-mcc-03 发送资源释放通知(指令ID, 资源用量)
                    销毁沙箱实例(上下文.沙箱实例)
                    从活跃沙箱表中移除(指令ID)
                    SET internal_state = STATE_WAIT

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报执行状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 不支持的语言类型 | 立即返回失败，错误码=UNSUPPORTED_LANGUAGE | — |
| 沙箱镜像拉取失败 | 重试1次，仍失败则返回失败，错误码=SANDBOX_IMAGE_ERROR | 镜像仓库恢复正常 |
| 代码编译/语法错误 | 收集标准错误输出，返回失败，错误码=COMPILE_ERROR | — |
| 沙箱进程异常崩溃 | 收集核心转储信息，返回失败，错误码=SANDBOX_CRASH | — |
| 输出数据超过大小上限 | 截断至上限，在结果中标记"输出已截断" | — |
| 检测到沙箱逃逸尝试 | 立即强制终止，发送安全违规告警，将该语言运行时标记为"需审查" | 人工审查后解除 |
| 紧急熔断 | 立即终止所有活跃沙箱实例，清空活跃沙箱表 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 代码执行指令 | 只读 | ag-mcc-01 发送 |
| 内部调度总线 | 读 | 语言运行时配置 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 读 | 资源约束参数 | 只读 | ag-mcc-03 随指令下发 |
| 内部调度总线 | 写 | 代码执行结果 | 专属写入 | 向 ag-mcc-01 返回 |
| 内部调度总线 | 写 | 资源释放通知 | 专属写入 | 向 ag-mcc-03 发送 |
| 内部调度总线 | 写 | 安全违规告警 | 专属写入 | 向 ag-mcc-01（转 ag-ecc-04）发送 |
| 内部调度总线 | 写 | 执行状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 所有代码必须在沙箱中执行，沙箱外不得执行任何用户或系统生成的代码 |
| S-02 | 网络访问能力默认完全禁止，仅可通过白名单审评后为特定工具开放 |
| S-03 | 文件系统写操作默认禁止，仅允许写入指定的临时目录，且执行完成后自动清除 |
| S-04 | 检测到沙箱逃逸尝试或禁止的系统调用时，必须立即终止沙箱并上报安全违规告警 |
| S-05 | 沙箱实例之间完全隔离，一个沙箱的崩溃不得影响其他沙箱或宿主系统 |
| S-06 | 代码执行结果中的标准输出/错误不得包含宿主系统的敏感信息（路径、配置等） |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M07-01 | `WAITING_COMMAND`，正常 Python 代码 | 代码执行指令（语言=Python，代码="print(2+3)"） | 沙箱执行成功，标准输出="5"，返回成功 |
| TC-M07-02 | `WAITING_COMMAND`，代码超时 | 代码执行指令（代码="while True: pass"，超时=5秒） | 5 秒后强制终止，返回超时状态 |
| TC-M07-03 | `WAITING_COMMAND`，内存溢出 | 代码执行指令（代码="a = [0]*10**9"，内存上限=64MB） | 进程被 OOM 终止，返回内存溢出 |
| TC-M07-04 | `WAITING_COMMAND`，语法错误 | 代码执行指令（语言=Python，代码="print(）"） | 捕获 SyntaxError，返回失败，错误码=COMPILE_ERROR |
| TC-M07-05 | `WAITING_COMMAND`，尝试网络访问 | 代码执行指令（代码="import socket; socket.connect(...)"） | 沙箱拦截，返回安全违规 |
| TC-M07-06 | `WAITING_COMMAND`，不支持的语言类型 | 代码执行指令（语言=Ruby） | 返回失败，错误码=UNSUPPORTED_LANGUAGE |
