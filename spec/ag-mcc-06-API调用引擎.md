# ag-mcc-06-API调用引擎 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-06 |
| 模块名称 | API调用引擎 |
| 所属分区 | 三、调用执行引擎 |
| 核心职责 | 作为 MCC 行动执行层中负责执行外部 API 调用的专属模块，接收 ag-mcc-01（执行调度核心）下发的 API 类型工具调用指令，根据 ag-mcc-04（工具注册中心）中存储的端点配置、认证方式与请求模板，构建标准的 HTTP 请求并发送至外部服务。管理请求超时、重试策略、响应解析与错误处理。将执行结果（成功响应数据或失败错误码）返回至 ag-mcc-01。不参与工具选择或参数决策，仅负责 API 请求的构建、发送与响应处理 |
| 依赖模块 | ag-mcc-01（执行调度核心，下发 API 调用指令）、ag-mcc-04（工具注册中心，获取 API 端点与认证配置）、ag-mcc-05（工具参数校验器，预校验参数已通过） |
| 被依赖模块 | ag-mcc-01（返回 API 调用执行结果）、ag-mcc-03（发送资源释放通知） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待指令 | `WAITING_COMMAND` | 无待执行的 API 调用，等待 ag-mcc-01 下发 | 系统初始化完成，活跃请求数为 0 |
| 请求发送中 | `REQUESTING` | 已构建请求并发送至外部服务，等待响应 | 收到 API 调用指令并完成请求构建 |
| 重试等待 | `RETRYING` | 上次请求失败且满足重试条件，正在等待后重试 | 请求失败且错误码属于可重试类型 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| API 调用指令 | Struct（指令ID + 步骤ID + 计划ID + 工具名称 + 参数列表 + 超时时间 + 安全令牌 + 重试策略覆盖） | ag-mcc-01 执行调度核心（通过内部调度总线） | ECC 下发工具调用且类型为 API 时 | **高** |
| API 端点配置 | Struct（工具名称 + 请求方法 + URL模板 + 请求头模板 + 认证方式 + 认证凭证引用 + 响应格式） | ag-mcc-04 工具注册中心 | 每次接收到 API 调用指令时查询 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| API 调用执行结果 | Struct（指令ID + 执行状态：成功/失败/超时 + 响应数据 + HTTP状态码 + 错误码 + 耗时 + 重试次数） | ag-mcc-01 执行调度核心（通过内部调度总线） | API 调用完成（成功或最终失败）时 | **高** |
| 资源释放通知 | Struct（指令ID + 释放的API次数 + 释放的Token数） | ag-mcc-03 资源配额管控单元 | 调用完成时 | 普通 |
| 调用状态上报 | Struct（当前状态 + 活跃请求数 + 今日调用次数 + 成功率 + 平均响应耗时） | ag-mcc-12 执行日志记录单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 请求配置与重试策略

### 一、认证方式实现

| 认证方式 | 凭证来源 | 请求中的携带位置 | 凭证刷新策略 |
|----------|------|:---:|------|
| API_KEY | 工具注册中心的认证凭证引用 | Header: `X-API-Key` | 固定值，过期前手动更新 |
| OAUTH2 | 认证服务的 Token 端点 | Header: `Authorization: Bearer {token}` | 自动检测 401 后刷新 |
| TOKEN | 工具注册中心的认证凭证引用 | Header: `Authorization: Token {token}` | 固定值，过期前手动更新 |
| NONE | — | — | 无需认证 |

### 二、重试策略

| 错误类型 | 是否重试 | 最大重试次数 | 重试间隔 | 重试间隔递增 |
|----------|:---:|:---:|:---:|:---:|
| 网络超时 (TIMEOUT) | 是 | 3 次 | 1 秒 | 指数退避（1s → 2s → 4s） |
| 服务暂时不可用 (503) | 是 | 3 次 | 2 秒 | 指数退避（2s → 4s → 8s） |
| 请求频率限制 (429) | 是 | 2 次 | 按 Retry-After 头 | 按服务端指示 |
| 客户端错误 (4xx，除 429) | 否 | 0 | — | — |
| 服务端错误 (5xx，除 503) | 否 | 0 | — | — |

### 三、超时配置

| 超时类型 | 默认值（秒） | 说明 |
|----------|:---:|------|
| 连接超时 | 10 | 建立 TCP 连接的最大等待时间 |
| 读取超时 | 30 | 等待服务器响应的最大时间 |
| 总超时 | 60 | 整个请求的硬性超时上限 |


## 核心处理逻辑

```
FUNCTION api_engine_main_loop():
    STATE_WAIT = WAITING_COMMAND
    STATE_REQ = REQUESTING
    STATE_RETRY = RETRYING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化活跃请求表 = {}  // 指令ID → 请求上下文

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收 API 调用指令
        IF 收到 ag-mcc-01 API 调用指令:
            SET internal_state = STATE_REQ
            指令 = API调用指令

            // 2a. 获取 API 端点配置
            端点配置 = 向 ag-mcc-04 查询 API 端点配置(指令.工具名称)
            IF 端点配置 == None:
                向 ag-mcc-01 返回执行结果(失败, "API端点配置不存在")
                SET internal_state = STATE_WAIT
                CONTINUE

            // 2b. 构建 HTTP 请求
            请求 = 构建HTTP请求(端点配置, 指令.参数列表)

            // 2c. 注入认证凭证
            请求 = 注入认证凭证(请求, 端点配置.认证方式)

            // 2d. 发送请求
            活跃请求表[指令.指令ID] = {
                指令: 指令,
                端点配置: 端点配置,
                重试次数: 0,
                开始时间: NOW()
            }
            发送异步HTTP请求(请求, 指令.指令ID, 指令.超时时间)

        // 第3步：处理 HTTP 响应
        IF 收到 HTTP 响应(指令ID):
            上下文 = 活跃请求表[指令ID]
            响应 = HTTP响应

            IF 响应.状态码 IN [200, 201, 204]:
                // 成功
                结果 = 解析成功响应(响应, 上下文.端点配置.响应格式)
                向 ag-mcc-01 返回执行结果(成功, 结果)
                向 ag-mcc-03 发送资源释放通知(指令ID)
                从活跃请求表中移除(指令ID)
                SET internal_state = STATE_WAIT

            ELSE IF 响应.状态码 IN [429, 503] AND 上下文.重试次数 < 最大重试次数:
                // 需要重试
                SET internal_state = STATE_RETRY
                上下文.重试次数++
                重试间隔 = 计算重试间隔(响应.状态码, 上下文.重试次数)
                计划在 重试间隔 秒后重新发送请求

            ELSE IF 响应.状态码 IN [401, 403] AND 上下文.端点配置.认证方式 == "OAUTH2":
                // Token 过期，尝试刷新
                刷新OAuth2Token(上下文.端点配置)
                上下文.重试次数++
                重新发送请求

            ELSE:
                // 不可重试的错误
                向 ag-mcc-01 返回执行结果(失败, 响应.状态码, 响应.错误信息)
                向 ag-mcc-03 发送资源释放通知(指令ID)
                从活跃请求表中移除(指令ID)
                SET internal_state = STATE_WAIT

        // 第4步：处理请求超时
        IF 检测到请求超时(指令ID):
            上下文 = 活跃请求表[指令ID]
            IF 上下文.重试次数 < 最大重试次数 AND 超时属于可重试类型:
                SET internal_state = STATE_RETRY
                上下文.重试次数++
                重试间隔 = 计算重试间隔("TIMEOUT", 上下文.重试次数)
                计划在 重试间隔 秒后重新发送请求
            ELSE:
                向 ag-mcc-01 返回执行结果(超时)
                向 ag-mcc-03 发送资源释放通知(指令ID)
                从活跃请求表中移除(指令ID)
                SET internal_state = STATE_WAIT

        // 第5步：执行重试
        IF internal_state == STATE_RETRY AND 重试等待时间已到:
            重新发送HTTP请求(上下文)
            SET internal_state = STATE_REQ

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报调用状态

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| API 端点配置不存在 | 立即返回失败，错误码=ENDPOINT_NOT_FOUND | 工具在 ag-mcc-04 中完成注册 |
| DNS 解析失败 | 视为网络错误，进入重试流程 | DNS 恢复正常 |
| 响应数据格式与预期不符 | 尝试降级解析，若失败则返回 PARSE_ERROR | — |
| OAuth2 Token 刷新失败 | 返回认证失败，错误码=AUTH_REFRESH_FAILED | Token 端点恢复 |
| 并发请求数达到上限 | 将新指令加入等待队列，按优先级排序 | 有请求完成释放槽位 |
| 紧急熔断 | 中断所有进行中的请求，清空等待队列 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | API 调用指令 | 只读 | ag-mcc-01 发送 |
| 内部调度总线 | 读 | API 端点配置 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 写 | API 调用执行结果 | 专属写入 | 向 ag-mcc-01 返回 |
| 内部调度总线 | 写 | 资源释放通知 | 专属写入 | 向 ag-mcc-03 发送 |
| 内部调度总线 | 写 | 调用状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| A-01 | API 调用引擎不得在日志或返回结果中暴露完整的认证凭证（API Key、Token 等），仅记录认证方式 |
| A-02 | 所有外部 API 请求必须通过 HTTPS 发送，拒绝明文 HTTP 请求 |
| A-03 | 响应数据在返回至 ag-mcc-01 前必须校验数据大小，超过上限（默认 10MB）的数据截断并标记 |
| A-04 | OAuth2 Token 刷新过程必须在独立的安全上下文中执行，刷新后的 Token 不得暴露给调用方 |
| A-05 | 重试请求必须携带幂等键，防止因网络抖动导致的重复执行副作用 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M06-01 | `WAITING_COMMAND`，正常 API 调用 | API 调用指令（工具=weather_api，参数={city:"北京"}，端点配置存在） | 发送 HTTPS 请求，收到 200 响应后返回成功结果 |
| TC-M06-02 | `WAITING_COMMAND`，收到 503 触发重试 | API 调用指令 + HTTP 响应（状态码=503） | 进入 RETRYING 状态，2 秒后重试，最多 3 次 |
| TC-M06-03 | `WAITING_COMMAND`，收到 429 按服务端指示重试 | API 调用指令 + HTTP 响应（状态码=429，Retry-After=5） | 5 秒后重试，最多 2 次 |
| TC-M06-04 | `WAITING_COMMAND`，收到 400 不重试 | API 调用指令 + HTTP 响应（状态码=400） | 立即返回失败，错误码=400 |
| TC-M06-05 | `REQUESTING`，请求超时进入重试 | 请求超时（未收到响应），重试次数=0 | 进入 RETRYING，1 秒后重试 |
| TC-M06-06 | `REQUESTING`，重试耗尽返回失败 | 请求超时，重试次数已达上限 3 | 返回执行结果（超时），释放资源 |
