# ag-mcc-09-结果校验器 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mcc-09 |
| 模块名称 | 结果校验器 |
| 所属分区 | 四、反馈与日志 |
| 核心职责 | 在各执行模块（API调用引擎、代码执行沙箱、文件操作执行器）返回执行结果后，对结果的格式合规性、字段完整性、数据合理性与安全合规性进行多维度校验。确保返回给 ECC 认知大脑的数据符合预期的结构化规范，过滤掉异常数据、截断超大数据、标记不完整结果。校验通过的结果放行至闭环反馈单元（ag-mcc-11）进行最终汇总；校验不通过的结果标记错误原因后仍返回，但附带详细的校验失败报告。不参与工具的实际调用或数据修改，仅负责结果质量的客观校验 |
| 依赖模块 | ag-mcc-01（执行调度核心，提供待校验的执行结果与预期输出定义）、ag-mcc-04（工具注册中心，获取各工具的预期响应格式模板） |
| 被依赖模块 | ag-mcc-11（闭环反馈单元，接收校验后的结果进行汇总）、ag-mcc-12（执行日志记录单元，记录校验失败事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待结果 | `WAITING_RESULT` | 无待校验的执行结果，等待调度核心下发 | 系统初始化完成，校验队列为空 |
| 校验进行中 | `VALIDATING` | 正在对执行结果进行多维度校验 | 收到执行结果校验请求 |
| 校验完成 | `VALIDATED` | 校验完成，已输出带校验标记的结果 | 全部校验维度检查完毕 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 执行结果校验请求 | Struct（指令ID + 步骤ID + 计划ID + 工具名称 + 工具类型 + 执行状态 + 原始输出数据 + 执行耗时 + 错误码） | ag-mcc-01 执行调度核心（通过内部调度总线） | 每次工具调用执行完成时 | **高** |
| 预期响应格式模板 | Struct（工具名称 + 期望的字段列表 + 各字段类型约束 + 允许的数据范围 + 最大响应大小 + 必填字段标记） | ag-mcc-04 工具注册中心 | 每次校验时查询 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mcc-01 执行调度核心 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 校验后执行结果 | Struct（指令ID + 步骤ID + 计划ID + 原始执行状态 + 校验标记：通过/格式异常/字段缺失/数据截断/安全告警 + 校验详情 + 原始输出数据 + 清洗后输出数据） | ag-mcc-11 闭环反馈单元 | 校验完成时 | **高** |
| 校验失败通知 | Struct（指令ID + 失败维度 + 详细错误列表 + 建议处理方式） | ag-mcc-12 执行日志记录单元 | 校验不通过时 | 普通 |
| 安全告警 | Struct（指令ID + 告警类型 + 数据特征 + 严重等级） | ag-mcc-01（转至 ag-ecc-04 安全仲裁模块） | 检测到异常数据模式时 | **高** |
| 校验状态上报 | Struct（当前状态 + 今日校验次数 + 通过率 + 各失败维度分布 + 平均校验耗时） | ag-mcc-12 执行日志记录单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 校验维度与规则

### 一、通用校验维度

| 校验维度 | 权重 | 判定方式 | 不通过时的标记 |
|----------|:---:|------|:---:|
| 格式合规性 | 0.30 | 输出数据结构与预期响应格式模板的匹配程度 | `FORMAT_ERROR` |
| 字段完整性 | 0.25 | 必填字段是否全部存在且非空 | `FIELD_MISSING` |
| 数据合理性 | 0.20 | 字段值是否在允许的数据范围内（类型、长度、取值范围） | `DATA_INVALID` |
| 大小合规性 | 0.10 | 响应数据大小是否在最大响应大小限制内 | `DATA_TRUNCATED` |
| 安全合规性 | 0.15 | 输出数据中是否包含敏感信息或异常模式 | `SECURITY_ALERT` |

### 二、各工具类型的专项校验

| 工具类型 | 专项校验项 | 说明 |
|:---:|------|------|
| API 调用 | HTTP 状态码校验 | 验证状态码是否在预期范围内（2xx 为正常） |
| API 调用 | JSON 格式校验 | 响应体是否为合法的 JSON（若预期为 JSON 格式） |
| 代码执行 | 输出大小校验 | 标准输出 + 标准错误的总大小是否超过上限 |
| 代码执行 | 退出码校验 | 退出码是否在允许的范围内 |
| 文件操作 | 文件哈希校验 | 写入操作后校验文件哈希是否与预期一致 |
| 文件操作 | 文件存在性校验 | 读取/删除操作后校验目标文件是否如预期存在/不存在 |

### 三、安全合规检查

| 检查项 | 说明 | 处理方式 |
|------|------|------|
| 敏感信息检测 | 输出数据中是否包含疑似密钥、Token、密码等敏感字符串 | 标记 SECURITY_ALERT，脱敏后放行 |
| 异常数据模式 | 输出数据中是否包含 SQL 注入特征、XSS 脚本等攻击载荷 | 标记 SECURITY_ALERT，发送安全告警 |
| 数据量异常 | 单次返回数据是否超过历史平均值的 500% | 标记 DATA_ANOMALY，截断至上限 |


## 核心处理逻辑

```
FUNCTION result_validator_main_loop():
    STATE_WAIT = WAITING_RESULT
    STATE_VALIDATE = VALIDATING
    STATE_DONE = VALIDATED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化校验统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收执行结果校验请求
        IF 收到 ag-mcc-01 执行结果校验请求:
            SET internal_state = STATE_VALIDATE
            请求 = 校验请求
            校验标记列表 = []
            清洗后数据 = 请求.原始输出数据

            // 2a. 获取预期响应格式模板
            预期模板 = 向 ag-mcc-04 查询预期响应格式模板(请求.工具名称)
            IF 预期模板 == None:
                // 无模板时执行通用校验
                预期模板 = 通用默认模板

            // 2b. 格式合规性校验
            IF NOT 格式匹配(请求.原始输出数据, 预期模板):
                校验标记列表.append("FORMAT_ERROR")

            // 2c. 字段完整性校验
            缺失字段列表 = 检查必填字段(请求.原始输出数据, 预期模板.必填字段)
            IF 缺失字段列表 非空:
                校验标记列表.append("FIELD_MISSING")
                校验详情["缺失字段"] = 缺失字段列表

            // 2d. 数据合理性校验
            无效字段列表 = 检查数据范围(请求.原始输出数据, 预期模板.字段约束)
            IF 无效字段列表 非空:
                校验标记列表.append("DATA_INVALID")
                校验详情["无效字段"] = 无效字段列表

            // 2e. 大小合规性校验
            IF 数据大小 > 预期模板.最大响应大小:
                清洗后数据 = 截断数据(请求.原始输出数据, 预期模板.最大响应大小)
                校验标记列表.append("DATA_TRUNCATED")

            // 2f. 安全合规性校验
            安全扫描结果 = 执行安全合规扫描(请求.原始输出数据)
            IF 安全扫描结果.发现敏感信息:
                清洗后数据 = 脱敏处理(清洗后数据, 安全扫描结果)
                校验标记列表.append("SECURITY_ALERT")
                向 ag-mcc-01 发送安全告警(请求.指令ID, 安全扫描结果)
            IF 安全扫描结果.发现异常模式:
                校验标记列表.append("SECURITY_ALERT")
                向 ag-mcc-01 发送安全告警(请求.指令ID, 安全扫描结果)

            // 2g. 输出校验结果
            SET internal_state = STATE_DONE

            IF 校验标记列表为空:
                校验标记 = "通过"
            ELSE:
                校验标记 = JOIN(校验标记列表, "|")

            校验后结果 = 构建校验后执行结果(请求, 校验标记, 清洗后数据, 校验详情)

            // 发送至闭环反馈单元
            向 ag-mcc-11 发送校验后执行结果(校验后结果)

            // 校验不通过时记录日志
            IF 校验标记 != "通过":
                向 ag-mcc-12 发送校验失败通知(请求.指令ID, 校验标记, 校验详情)

            // 更新统计
            更新校验统计(校验标记)

            SET internal_state = STATE_WAIT

        // 第3步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mcc-12 上报校验状态

        SLEEP 10ms


FUNCTION 执行安全合规扫描(输出数据):
    扫描结果 = {发现敏感信息: False, 发现异常模式: False, 详情: []}

    // 检测敏感信息模式
    敏感模式列表 = [
        "sk-[a-zA-Z0-9]{32,}",   // API Key 模式
        "Bearer [a-zA-Z0-9_-]+", // Token 模式
        "password", "secret", "private_key"
    ]
    FOR EACH 模式 IN 敏感模式列表:
        IF 正则匹配(输出数据, 模式):
            扫描结果.发现敏感信息 = True
            扫描结果.详情.append("疑似敏感信息: 匹配模式=" + 模式)

    // 检测异常数据模式
    异常模式列表 = [
        "(?i)drop\s+table",       // SQL 注入
        "(?i)<script.*?>",        // XSS 脚本
        "(?i)exec\s*\(.*\)",      // 命令注入
    ]
    FOR EACH 模式 IN 异常模式列表:
        IF 正则匹配(输出数据, 模式):
            扫描结果.发现异常模式 = True
            扫描结果.详情.append("疑似攻击载荷: 匹配模式=" + 模式)

    RETURN 扫描结果
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 预期响应格式模板不存在 | 使用通用默认模板（仅检查数据格式是否为合法的 JSON/纯文本），标记"无模板" | 工具注册中心补充该工具的模板 |
| 输出数据完全为空 | 标记 FIELD_MISSING，返回空的清洗后数据 | — |
| 输出数据格式无法识别 | 标记 FORMAT_ERROR，将原始数据作为纯文本保留 | — |
| 数据截断后导致 JSON 格式破坏 | 截断点回溯至上一个完整 JSON 结构边界 | — |
| 同一指令重复校验 | 基于指令ID去重，5秒内相同指令ID直接返回缓存结果 | — |
| 紧急熔断 | 暂停校验，未完成的请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 执行结果校验请求 | 只读 | ag-mcc-01 发送 |
| 内部调度总线 | 读 | 预期响应格式模板 | 只读 | ag-mcc-04 提供 |
| 内部调度总线 | 写 | 校验后执行结果 | 专属写入 | 向 ag-mcc-11 发送 |
| 内部调度总线 | 写 | 校验失败通知 | 专属写入 | 向 ag-mcc-12 发送 |
| 内部调度总线 | 写 | 安全告警 | 专属写入 | 向 ag-mcc-01（转 ag-ecc-04）发送 |
| 内部调度总线 | 写 | 校验状态上报 | 周期性写入 | 向 ag-mcc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| V-01 | 本模块仅校验结果质量，不得修改工具返回的原始业务数据内容（脱敏与截断除外） |
| V-02 | 敏感信息检测为强制性安全检查，发现敏感信息必须脱敏后方可放行 |
| V-03 | 检测到攻击载荷模式时，必须立即发送安全告警，不得仅标记后放行 |
| V-04 | 校验过程中不得将原始输出数据缓存或转发至任何非授权的第三方模块 |
| V-05 | 校验失败的详细原因仅记录于内部日志，返回给 ECC 的错误信息应简洁明了，不得暴露系统内部实现细节 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M09-01 | `WAITING_RESULT`，正常结果 | 执行结果（状态=成功，输出数据完全符合预期模板） | 校验标记=通过，清洗后数据=原始数据 |
| TC-M09-02 | `WAITING_RESULT`，缺少必填字段 | 执行结果（输出数据缺少模板中标记为 required 的字段） | 校验标记=FIELD_MISSING，校验详情列出缺失字段 |
| TC-M09-03 | `WAITING_RESULT`，数据超出大小上限 | 执行结果（输出数据大小=150MB，上限=100MB） | 校验标记=DATA_TRUNCATED，清洗后数据被截断 |
| TC-M09-04 | `WAITING_RESULT`，检测到敏感信息 | 执行结果（输出数据包含 "sk-abc123..."） | 校验标记=SECURITY_ALERT，脱敏后放行，发送安全告警 |
| TC-M09-05 | `WAITING_RESULT`，格式不匹配 | 执行结果（预期JSON格式，实际返回纯文本） | 校验标记=FORMAT_ERROR |
| TC-M09-06 | `WAITING_RESULT`，工具无模板 | 执行结果（工具未注册响应模板） | 使用通用默认模板校验，标记"无模板" |
